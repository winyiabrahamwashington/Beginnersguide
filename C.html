What is C#?

The powerful, modern programming language from Microsoft

The C# Programming Language

C# (pronounced "C Sharp") is a modern, object-oriented programming language developed by Microsoft in 2000 as part of its .NET initiative. It was created by Anders Hejlsberg and his team with the goal of combining the computing power of C++ with the programming ease of Visual Basic.

Key Features of C#

  • Object-Oriented: Pure OOP language with support for encapsulation, inheritance, and polymorphism
  • Type-Safe: Prevents unsafe type conversions and provides memory safety through garbage collection
  • Modern Language Features: Includes properties, LINQ, async/await, lambda expressions, and more
  • Cross-Platform: With .NET Core (now .NET 5+), C# can run on Windows, Linux, and macOS
  • Rich Ecosystem: Extensive standard library and NuGet package ecosystem
  • Versatile: Used for web, desktop, mobile, cloud, gaming, IoT, and AI applications
  • Strongly Typed: Variables must be declared with specific types
  • Automatic Memory Management: Garbage collector handles memory allocation and deallocation

Understanding the .NET Ecosystem

C# is part of the larger .NET platform which consists of several key components:

.NET Platform Components

  • .NET SDK: Contains tools for developing .NET applications (compiler, CLI tools, etc.)
  • .NET Runtime: Needed to run .NET applications (includes CLR and core libraries)
  • Common Language Runtime (CLR): Executes C# intermediate language (IL) code
  • Base Class Library (BCL): Extensive collection of ready-made classes for common tasks
  • ASP.NET Core: Framework for building web applications
  • Entity Framework: Object-Relational Mapping (ORM) for database access
  • NuGet: Package manager for .NET libraries

How C# Works: C# source code (.cs files) is compiled by the Roslyn compiler into intermediate language (IL) code. The CLR then compiles this IL code into native machine code at runtime using Just-In-Time (JIT) compilation.

// Simple C# program
using System;

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Hello, C#!");
        
        // Variables
        string name = "Alice";
        int age = 30;
        
        // String interpolation
        Console.WriteLine($"My name is {name} and I'm {age} years old.");
    }
}

Did you know? C# was originally called "Cool" (C-like Object Oriented Language) during development. The name was changed to C# before release, with the # symbol representing four plus signs (++++) to indicate it was an evolution beyond C++.

Getting Started with C#

Set up your development environment and write your first program

Installing .NET SDK

To start programming in C#, you'll need to install the .NET SDK which includes the runtime and developer tools.

Installation Steps

  1. Download the latest .NET SDK from Microsoft's website
  2. Run the installer for your operating system
  3. Verify installation by running dotnet --version in your terminal
  4. Optionally install Visual Studio or VS Code for a better development experience

SDK vs Runtime

The .NET SDK is for developers (contains compiler and tools) while the .NET Runtime is for running .NET applications. When installing for development, always choose the SDK.

Beginner Tip

If you're using Windows, Visual Studio provides the most complete C# development experience. On macOS/Linux or for a lighter option, use VS Code with the C# extension.

Your First C# Program

Let's create the traditional "Hello World" program to understand C#'s basic structure.

using System;

class Program
{
    static void Main(string[] args)
    {
        Console.WriteLine("Hello, World!");
    }
}

Program Breakdown

  • using System;: Imports the System namespace for basic functionality
  • class Program: Defines a class named Program
  • static void Main(string[] args): The Main method - program entry point
  • Console.WriteLine(): Prints output to the console
  • //: Single-line comment
  • /* */: Multi-line comment
  • ///: XML documentation comment

Important: The Main method must be static and is case-sensitive. In .NET 6+ with top-level statements, the Main method is generated automatically.

Compiling and Running

To compile and run your program:

  1. Save the file as Program.cs
  2. Open terminal in the file's directory
  3. Run: dotnet run (for .NET projects)
  4. Or compile with: csc Program.cs (Windows only, using CSC compiler)

You should see "Hello, World!" printed in your terminal.

Variables and Data Types

C# is a statically-typed language, meaning variables must be declared with a specific type.

// Value types (stored on stack)
byte age = 30;                  // 8-bit unsigned integer (0 to 255)
short year = 2023;              // 16-bit integer (-32,768 to 32,767)
int population = 8000000;       // 32-bit integer (-2^31 to 2^31-1)
long globalPopulation = 7800000000L; // 64-bit integer (add L suffix)
float price = 19.99f;           // 32-bit floating point (add f suffix)
double pi = 3.1415926535;       // 64-bit floating point (default for decimals)
decimal money = 100.50m;        // 128-bit precise decimal (add m suffix)
char grade = 'A';               // Single 16-bit Unicode character
bool isCSharpFun = true;        // true or false

// Reference types (stored on heap)
string name = "C# Beginner";    // Sequence of characters
int[] numbers = {1, 2, 3};      // Array of integers
object obj = new object();      // Any C# object

// Constants (readonly variables)
const double PI = 3.14159;

// Type inference with var
var message = "Hello";          // Compiler infers string
var number = 42;                // Compiler infers int

// Nullable types (value types that can be null)
int? nullableInt = null;

Naming Conventions: Use camelCase for variables (myVariableName), PascalCase for classes and methods (MyClassName), and UPPER_CASE for constants (MAX_VALUE). Choose descriptive names that indicate the variable's purpose.

Beginner Tip

Start with int for whole numbers, double for decimal numbers, and string for text unless you have specific requirements. These are the most commonly used types in C#.

Type Casting

C# supports both implicit (safe) and explicit (manual) type conversions:

// Implicit conversion (safe, no data loss)
int myInt = 9;
double myDouble = myInt;  // Automatic casting: int to double

// Explicit conversion (manual, potential data loss)
double myDouble = 9.78;
int myInt = (int)myDouble; // Manual casting: double to int

// Conversion methods
string strNumber = "123";
int parsedInt = int.Parse(strNumber); // String to int
string intString = 123.ToString();    // Int to string

Operators

C# provides various operators for performing operations on variables and values.

// Arithmetic operators
int sum = 10 + 20;      // Addition
int difference = 20 - 10; // Subtraction
int product = 10 * 20;  // Multiplication
int quotient = 20 / 10; // Division
int remainder = 11 % 3; // Modulus (remainder)

// Assignment operators
int x = 10;
x += 5;  // Equivalent to x = x + 5
x -= 3;  // Equivalent to x = x - 3
x *= 2;  // Equivalent to x = x * 2
x /= 4;  // Equivalent to x = x / 4

// Comparison operators
bool isEqual = (10 == 20);    // Equal to
bool isNotEqual = (10 != 20); // Not equal to
bool isGreater = (10 > 20);   // Greater than
bool isLess = (10 < 20);      // Less than
bool isGreaterOrEqual = (10 >= 20);

// Logical operators
bool andResult = (true && false); // Logical AND
bool orResult = (true || false);  // Logical OR
bool notResult = !true;           // Logical NOT

// Ternary operator
int time = 20;
string result = (time < 18) ? "Good day." : "Good evening.";

// Null-coalescing operator
string name = null;
string displayName = name ?? "Guest"; // If name is null, use "Guest"

Operator Precedence

C# evaluates expressions according to operator precedence (order of operations):

  1. Parentheses ()
  2. Unary operators ++ -- !
  3. Multiplicative * / %
  4. Additive + -
  5. Relational < > <= >=
  6. Equality == !=
  7. Logical AND &&
  8. Logical OR ||
  9. Assignment = += -= etc.

Common Operator Mistakes

  • Using = (assignment) instead of == (equality)
  • Forgetting that integer division truncates decimals
  • Not understanding operator precedence
  • Using & (bitwise AND) instead of && (logical AND)
  • Confusing prefix (++x) and postfix (x++) increment

Control Flow Statements

Control flow statements allow you to control the execution flow of your program.

// If-else statement
int age = 18;
if (age < 13) {
    Console.WriteLine("Child");
} else if (age < 20) {
    Console.WriteLine("Teenager");
} else {
    Console.WriteLine("Adult");
}

// Switch statement (C# 8+ enhanced)
string day = "Monday";
switch (day) {
    case "Monday":
        Console.WriteLine("Start of work week");
        break;
    case "Friday":
        Console.WriteLine("End of work week");
        break;
    default:
        Console.WriteLine("Midweek day");
        break;
}

// Switch expression (C# 8+)
string message = day switch {
    "Monday" => "Start of work week",
    "Friday" => "End of work week",
    _ => "Midweek day"
};

// Traditional for loop
for (int i = 0; i < 5; i++) {
    Console.WriteLine("Iteration: " + i);
}

// Foreach loop
int[] numbers = {1, 2, 3, 4, 5};
foreach (int num in numbers) {
    Console.WriteLine(num);
}

// While loop
int count = 0;
while (count < 5) {
    Console.WriteLine("Count: " + count);
    count++;
}

// Do-while loop
int num = 1;
do {
    Console.WriteLine("Number: " + num);
    num++;
} while (num <= 5);

// Break and continue
for (int i = 0; i < 10; i++) {
    if (i == 4) {
        continue; // Skip iteration
    }
    if (i == 8) {
        break; // Exit loop
    }
    Console.WriteLine(i);
}

Foreach Loop: C# provides a simplified loop for iterating through collections: foreach (var item in collection) { ... }

Control Flow Tips

When working with control flow:

  • Always use braces {} even for single statements
  • Indent code properly for readability
  • Use break in switch statements to prevent fall-through
  • Consider using foreach loops when possible
  • Avoid deep nesting of control structures
  • Use meaningful conditions in loops and if statements

Control Flow Exercises

Try these exercises to practice control flow:

  1. Write a program that prints numbers 1-100, but for multiples of 3 print "Fizz", for multiples of 5 print "Buzz", and for multiples of both print "FizzBuzz"
  2. Create a calculator program that takes two numbers and an operator (+, -, *, /) as input and outputs the result
  3. Write a program that prints a pyramid pattern using nested loops

Object-Oriented Programming in C#

Learn the core concepts of OOP with C#

Classes and Objects

C# is fundamentally object-oriented. A class is a blueprint for objects, and an object is an instance of a class.

// Class definition
public class Car {
    // Fields (attributes)
    private string brand;
    private string model;
    private int year;
    
    // Constructor
    public Car(string brand, string model, int year) {
        this.brand = brand;
        this.model = model;
        this.year = year;
    }
    
    // Properties (getters/setters)
    public string Brand {
        get { return brand; }
        set { brand = value; }
    }
    
    public string Model {
        get => model;
        set => model = value;
    }
    
    public int Year { get; set; } // Auto-implemented property
    
    // Method
    public void StartEngine() {
        Console.WriteLine($"Starting the {brand} {model}");
    }
    
    // Method with return value
    public int CalculateAge(int currentYear) {
        return currentYear - year;
    }
}

// Creating and using objects
class Program {
    static void Main(string[] args) {
        // Create Car object
        Car myCar = new Car("Toyota", "Camry", 2020);
        
        // Call methods
        myCar.StartEngine();  // Output: Starting the Toyota Camry
        
        // Access properties
        Console.WriteLine("Brand: " + myCar.Brand);
        
        // Calculate car age
        int carAge = myCar.CalculateAge(2023);
        Console.WriteLine("Car age: " + carAge + " years");
    }
}

Class Components

  • Fields: Variables that hold the object's state (should usually be private)
  • Constructor: Special method for object initialization (same name as class)
  • Properties: Provide controlled access to fields (prefer over public fields)
  • Methods: Functions that define object behavior
  • this keyword: Refers to the current object
  • new keyword: Creates a new object instance

Best Practice: Follow the principle of encapsulation by making fields private and providing public properties to access them. This protects the object's internal state and allows for validation.

Object-Oriented Thinking

When designing classes:

  1. Identify real-world objects your program needs
  2. Determine each object's attributes (fields) and behaviors (methods)
  3. Define relationships between objects
  4. Keep classes focused on a single responsibility
  5. Use meaningful names that reflect the object's purpose

Four Pillars of OOP

Object-Oriented Programming in C# is built on four main principles:

1. Encapsulation

Bundling data and methods that operate on that data within one unit (class) and restricting access to some of the object's components.

public class BankAccount {
    private decimal balance;  // Private field
    
    // Public methods to interact with balance
    public void Deposit(decimal amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
    
    public void Withdraw(decimal amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
    
    public decimal GetBalance() {
        return balance;
    }
}

Benefits: Prevents unauthorized access, allows validation, makes code more maintainable

2. Inheritance

Creating new classes (derived classes) from existing ones (base classes) to reuse code and establish relationships.

// Base class
class Vehicle {
    protected string brand;  // Protected allows access by derived classes
    
    public Vehicle(string brand) {
        this.brand = brand;
    }
    
    public void Honk() {
        Console.WriteLine("Honk honk!");
    }
}

// Derived class
class Car : Vehicle {
    private string model;
    
    public Car(string brand, string model) : base(brand) {
        this.model = model;
    }
    
    public void DisplayInfo() {
        Console.WriteLine($"Brand: {brand}, Model: {model}");
    }
}

// Using inheritance
class Program {
    static void Main(string[] args) {
        Car myCar = new Car("Toyota", "Camry");
        myCar.Honk();       // Inherited method
        myCar.DisplayInfo(); // Derived class method
    }
}

Benefits: Code reuse, logical hierarchy, polymorphism

3. Polymorphism

The ability of an object to take on many forms, typically through method overriding and interfaces.

class Animal {
    public virtual void MakeSound() {
        Console.WriteLine("Some sound");
    }
}

class Dog : Animal {
    public override void MakeSound() {
        Console.WriteLine("Bark");
    }
}

class Cat : Animal {
    public override void MakeSound() {
        Console.WriteLine("Meow");
    }
}

// Using polymorphism
class Program {
    static void Main(string[] args) {
        Animal myAnimal = new Animal();  // Animal object
        Animal myDog = new Dog();        // Dog object
        Animal myCat = new Cat();        // Cat object
        
        myAnimal.MakeSound();  // Output: Some sound
        myDog.MakeSound();     // Output: Bark
        myCat.MakeSound();     // Output: Meow
    }
}

Benefits: Flexible code, easier to extend, cleaner interfaces

4. Abstraction

Hiding complex implementation details and showing only the necessary features of an object.

// Abstract class
abstract class Shape {
    // Abstract method (no implementation)
    public abstract double Area();
    
    // Concrete method
    public void Display() {
        Console.WriteLine("Area: " + Area());
    }
}

// Concrete derived class
class Circle : Shape {
    private double radius;
    
    public Circle(double r) { radius = r; }
    
    public override double Area() {
        return Math.PI * radius * radius;
    }
}

// Using abstraction
class Program {
    static void Main(string[] args) {
        Shape myCircle = new Circle(5.0);
        myCircle.Display();  // Output: Area: 78.53981633974483
    }
}

Benefits: Simplifies complex systems, reduces complexity, focuses on what rather than how

SOLID Principles: Beyond the four pillars, good OOP design follows SOLID principles:

  1. Single Responsibility: A class should have one reason to change
  2. Open-Closed: Open for extension, closed for modification
  3. Liskov Substitution: Derived classes should be substitutable for their base classes
  4. Interface Segregation: Many specific interfaces are better than one general one
  5. Dependency Inversion: Depend on abstractions, not concretions

Interfaces and Abstract Classes

Interfaces and abstract classes are key tools for implementing abstraction in C#.

// Interface definition
interface IDrawable {
    void Draw();  // Interface members are implicitly public
    
    // Default implementation (C# 8+)
    void Resize() {
        Console.WriteLine("Resizing");
    }
    
    // Static method (C# 8+)
    static void PrintInfo() {
        Console.WriteLine("This is a IDrawable interface");
    }
}

// Abstract class
abstract class Shape {
    protected string color;
    
    public Shape(string color) {
        this.color = color;
    }
    
    // Abstract method
    public abstract double Area();
    
    // Concrete method
    public string GetColor() {
        return color;
    }
}

// Concrete class implementing interface and extending abstract class
class Circle : Shape, IDrawable {
    private double radius;
    
    public Circle(string color, double radius) : base(color) {
        this.radius = radius;
    }
    
    public override double Area() {
        return Math.PI * radius * radius;
    }
    
    public void Draw() {
        Console.WriteLine($"Drawing a {color} circle");
    }
}

// Using interface and abstract class
class Program {
    static void Main(string[] args) {
        Circle myCircle = new Circle("red", 5.0);
        myCircle.Draw();          // Output: Drawing a red circle
        myCircle.Resize();        // Output: Resizing
        Console.WriteLine("Area: " + myCircle.Area());
        IDrawable.PrintInfo();    // Output: This is a IDrawable interface
    }
}

Interfaces vs Abstract Classes

Feature Interface Abstract Class
Methods Abstract (default/static in C# 8+) Can have both abstract and concrete
Fields Not allowed Can have any fields
Constructors No Yes
Multiple Inheritance Class can implement multiple Class can inherit only one
Access Modifiers Members are public by default Can have any access modifier

Interface Tips

When working with interfaces:

  • Use interfaces to define behavior contracts
  • A class can implement multiple interfaces
  • C# 8+ allows default and static methods in interfaces
  • Prefer interfaces to abstract classes when possible
  • Use the I prefix for interface names (e.g., IDrawable)

Modern C# Features: C# continues to evolve with features that make OOP more powerful:

  • Records (immutable data types)
  • Pattern matching
  • Init-only properties
  • Nullable reference types
  • Top-level statements

Collections and Generics

C# provides a rich set of collection types and generics for working with groups of data.

using System;
using System.Collections.Generic;

class CollectionsDemo {
    static void Main() {
        // List - ordered collection (allows duplicates)
        List fruits = new List();
        fruits.Add("Apple");
        fruits.Add("Banana");
        fruits.Insert(1, "Orange"); // Insert at index 1
        
        // Dictionary - key-value pairs
        Dictionary ages = new Dictionary();
        ages["John"] = 25;
        ages["Alice"] = 30;
        ages["Bob"] = 28;
        
        // HashSet - unordered collection of unique items
        HashSet uniqueNumbers = new HashSet();
        uniqueNumbers.Add(10);
        uniqueNumbers.Add(20);
        uniqueNumbers.Add(10); // Duplicate, won't be added
        
        // Queue - FIFO collection
        Queue queue = new Queue();
        queue.Enqueue("First");
        queue.Enqueue("Second");
        queue.Enqueue("Third");
        
        // Stack - LIFO collection
        Stack stack = new Stack();
        stack.Push("First");
        stack.Push("Second");
        stack.Push("Third");
        
        // Iterating through collections
        Console.WriteLine("Fruits:");
        foreach (string fruit in fruits) {
            Console.WriteLine(fruit);
        }
        
        Console.WriteLine("\nAges:");
        foreach (KeyValuePair entry in ages) {
            Console.WriteLine($"{entry.Key}: {entry.Value}");
        }
    }
}

Common Collection Types

  • List: Dynamic array (most commonly used collection)
  • Dictionary: Key-value pairs
  • HashSet: Collection of unique values
  • Queue: First-In-First-Out (FIFO) collection
  • Stack: Last-In-First-Out (LIFO) collection
  • LinkedList: Doubly-linked list
  • SortedDictionary: Sorted key-value pairs

Generics: C# collections use generics () to provide type safety. Always specify the type when declaring collections: List rather than non-generic ArrayList.

Exception Handling

C# provides a robust exception handling mechanism to deal with runtime errors.

using System;
using System.IO;

class ExceptionHandling {
    static void Main() {
        // Try-catch block
        try {
            int[] numbers = {1, 2, 3};
            Console.WriteLine(numbers[5]); // IndexOutOfRangeException
        } catch (IndexOutOfRangeException e) {
            Console.WriteLine("Array index out of bounds!");
        }
        
        // Multiple catch blocks
        try {
            File.ReadAllText("nonexistent.txt"); // FileNotFoundException
        } catch (FileNotFoundException e) {
            Console.WriteLine("File not found: " + e.Message);
        } catch (Exception e) {
            Console.WriteLine("Something went wrong");
        }
        
        // Finally block
        FileStream fileStream = null;
        try {
            fileStream = File.OpenRead("example.txt");
            // Work with file
        } catch (IOException e) {
            Console.WriteLine("IO error: " + e.Message);
        } finally {
            if (fileStream != null) {
                fileStream.Close(); // Always executed
            }
        }
        
        // Try-with-resources (using statement)
        try (FileStream autoCloseStream = File.OpenRead("example.txt")) {
            // Work with file
        } catch (IOException e) {
            Console.WriteLine("IO error: " + e.Message);
        } // Stream automatically closed
        
        // Throwing exceptions
        try {
            ValidateAge(15);
        } catch (InvalidAgeException e) {
            Console.WriteLine(e.Message);
        }
    }
    
    // Custom exception
    class InvalidAgeException : Exception {
        public InvalidAgeException(string message) : base(message) {}
    }
    
    // Method throwing exception
    static void ValidateAge(int age) {
        if (age < 18) {
            throw new InvalidAgeException("Age must be 18 or older");
        }
    }
}

Exception Hierarchy

  • Exception: Base class for all exceptions
  • SystemException: Base class for system exceptions
  • ApplicationException: Base class for application exceptions
  • IOException: For I/O operation errors
  • ArgumentException: For invalid arguments
  • NullReferenceException: When null object is accessed
  • IndexOutOfRangeException: When array index is invalid

Best Practices:

  • Use specific exception types rather than catching general Exception
  • Don't catch exceptions you can't handle
  • Clean up resources in finally blocks or use using statements
  • Document exceptions your methods throw with XML comments
  • Create meaningful custom exceptions when appropriate

File I/O and Serialization

C# provides comprehensive APIs for working with files and serialization.

using System;
using System.IO;
using System.Text.Json;

class FileIO {
    static void Main() {
        // Writing to a file
        try {
            File.WriteAllText("output.txt", "Hello, File I/O!\n");
            File.AppendAllText("output.txt", "This is a second line.\n");
        } catch (IOException e) {
            Console.WriteLine("Error writing to file: " + e.Message);
        }
        
        // Reading from a file
        try {
            string content = File.ReadAllText("output.txt");
            Console.WriteLine(content);
        } catch (IOException e) {
            Console.WriteLine("Error reading file: " + e.Message);
        }
        
        // Working with streams
        try {
            using (StreamWriter writer = new StreamWriter("stream.txt")) {
                writer.WriteLine("Line 1");
                writer.WriteLine("Line 2");
            }
            
            using (StreamReader reader = new StreamReader("stream.txt")) {
                string line;
                while ((line = reader.ReadLine()) != null) {
                    Console.WriteLine(line);
                }
            }
        } catch (IOException e) {
            Console.WriteLine("Stream error: " + e.Message);
        }
        
        // Object serialization (JSON)
        Person person = new Person("Alice", 30);
        string json = JsonSerializer.Serialize(person);
        File.WriteAllText("person.json", json);
        
        // Object deserialization
        string jsonFromFile = File.ReadAllText("person.json");
        Person deserializedPerson = JsonSerializer.Deserialize(jsonFromFile);
        Console.WriteLine($"Deserialized: {deserializedPerson.Name}, {deserializedPerson.Age}");
    }
}

// Serializable class
class Person {
    public string Name { get; set; }
    public int Age { get; set; }
    
    public Person(string name, int age) {
        Name = name;
        Age = age;
    }
}

Key I/O Concepts

  • File class: Static methods for file operations
  • Directory class: Static methods for directory operations
  • StreamReader/StreamWriter: For text file I/O
  • FileStream: For binary file I/O
  • Serialization: Converting objects to formats like JSON or XML
  • Deserialization: Converting back from formats to objects

Using Statement: Always use the using statement for I/O operations to ensure proper resource cleanup. This works with any class implementing IDisposable.

Asynchronous Programming

C# supports asynchronous programming with async/await for non-blocking operations.

using System;
using System.Net.Http;
using System.Threading.Tasks;

class AsyncDemo {
    static async Task Main() {
        Console.WriteLine("Starting async operations...");
        
        // Start multiple async operations
        Task downloadTask = DownloadWebsiteAsync("https://example.com");
        Task calculationTask = CalculateFactorialAsync(5);
        
        // Do other work while waiting
        Console.WriteLine("Working on other tasks...");
        
        // Wait for all tasks to complete
        string websiteContent = await downloadTask;
        int factorialResult = await calculationTask;
        
        Console.WriteLine($"Downloaded {websiteContent.Length} characters");
        Console.WriteLine($"Factorial result: {factorialResult}");
    }
    
    static async Task DownloadWebsiteAsync(string url) {
        using (HttpClient client = new HttpClient()) {
            Console.WriteLine($"Downloading {url}...");
            string result = await client.GetStringAsync(url);
            Console.WriteLine($"Finished downloading {url}");
            return result;
        }
    }
    
    static async Task CalculateFactorialAsync(int n) {
        return await Task.Run(() => {
            Console.WriteLine($"Calculating factorial of {n}...");
            int result = 1;
            for (int i = 2; i <= n; i++) {
                result *= i;
            }
            Console.WriteLine($"Finished calculating factorial of {n}");
            return result;
        });
    }
}

Async Concepts

  • async/await: Keywords for writing asynchronous code
  • Task: Represents an asynchronous operation
  • Task: Represents an operation that returns a value
  • Non-blocking: Async methods don't block the calling thread
  • Exception handling: Use try/catch with await
  • Task.Run: For CPU-bound operations

Async Best Practices: Avoid async void methods (except event handlers), always await Tasks, use ConfigureAwait(false) in library code, and don't block on async code with .Result or .Wait().

Modern C# Async Features: C# continues to enhance async programming with:

  • Async streams (IAsyncEnumerable)
  • ValueTask for performance optimization
  • Async disposables (IAsyncDisposable)
  • Enhanced pattern matching with async
