Getting Started with Go

Go (often referred to as Golang) is a statically typed, compiled programming language designed at Google by Robert Griesemer, Rob Pike, and Ken Thompson. It was announced in 2009 and is known for its simplicity, efficiency, and excellent support for concurrent programming.

Go is used for cloud services, distributed systems, networking tools, and more

Go was designed to address common criticisms of other languages while maintaining their useful characteristics. It combines the performance and safety of compiled languages like C++ with the readability and usability of interpreted languages like Python.

What You'll Need

  • Go installed on your computer (download from golang.org)
  • A text editor or IDE (VS Code with Go extension, GoLand, etc.)
  • Basic command line knowledge

Why Learn Go?

Go is an excellent language to learn because:

  • Simplicity: Clean syntax with minimal keywords
  • Performance: Compiled to native machine code
  • Concurrency: Built-in support for concurrent programming
  • Strong Standard Library: Rich set of built-in packages
  • Growing Popularity: Used by Google, Uber, Twitch, Docker, and more

Your First Go Program

Here's the classic "Hello, World!" program in Go:

package main import "fmt" func main() { fmt.Println("Hello, World!") }

To run this, save it as hello.go and execute it with go run hello.go in your terminal.

Variables and Data Types

Go is statically typed, meaning variables have a specific type that can't change. However, Go can infer types in many cases using the := syntax.

// Variable declaration examples var name string = "Alice" // Explicit type declaration var age = 25 // Type inference height := 5.9 // Short declaration (type inferred) isStudent := true // Boolean // Multiple variable declaration var ( firstName = "John" lastName = "Doe" ) // Constants const pi = 3.14159

Control Structures

Go has similar control structures to other C-like languages but with some simplifications.

Conditional Statements (if, else if, else)

// Basic if statement if x > 10 { fmt.Println("x is greater than 10") } // If with initialization if y := computeValue(); y > 10 { fmt.Println("y is greater than 10") } else if y > 5 { fmt.Println("y is greater than 5") } else { fmt.Println("y is 5 or less") }

Switch Statement

// Basic switch switch day { case "Monday": fmt.Println("Start of the week") case "Friday": fmt.Println("Almost weekend") default: fmt.Println("Midweek") } // Switch with no condition (like if-else chain) switch { case x < 0: fmt.Println("Negative") case x == 0: fmt.Println("Zero") default: fmt.Println("Positive") }

Loops (for is the only loop in Go)

// Basic for loop for i := 0; i < 5; i++ { fmt.Println(i) } // While-like loop j := 0 for j < 5 { fmt.Println(j) j++ } // Infinite loop for { fmt.Println("This will run forever") break // Unless we break! } // Range loop (for arrays, slices, maps) numbers := []int{1, 2, 3} for index, value := range numbers { fmt.Printf("Index: %d, Value: %d\n", index, value) }
Go doesn't have while or do-while loops. All looping is done with for statements.

Functions

Functions in Go can return multiple values and have named return values. They're declared with the func keyword.

// Basic function func greet(name string) { fmt.Printf("Hello, %s!\n", name) } // Function with return value func add(a, b int) int { return a + b } // Multiple return values func divide(a, b float64) (float64, error) { if b == 0.0 { return 0.0, fmt.Errorf("cannot divide by zero") } return a / b, nil } // Named return values func rectangleProps(length, width float64) (area, perimeter float64) { area = length * width perimeter = 2 * (length + width) return // naked return (returns named variables) } // Variadic function (accepts variable number of args) func sum(numbers ...int) int { total := 0 for _, num := range numbers { total += num } return total }

Function Calls

// Calling functions greet("Alice") result := add(5, 3) fmt.Println(result) // 8 quotient, err := divide(10.0, 2.0) if err != nil { fmt.Println(err) } else { fmt.Println(quotient) // 5 } a, p := rectangleProps(10.5, 5.5) fmt.Printf("Area: %.2f, Perimeter: %.2f\n", a, p) total := sum(1, 2, 3, 4, 5) fmt.Println(total) // 15
In Go, functions that start with a capital letter are exported (public) and can be used by other packages. Functions starting with lowercase letters are private to their package.

Packages and Imports

Go programs are organized into packages. The main package is special as it defines an executable program.

// Package declaration (first line of file) package main // Importing packages import ( "fmt" // Standard library package "math" // Another standard package "os" // Operating system functions "strconv" // String conversions "github.com/user/package" // External package ) // Import with alias import m "math" // Dot import (not recommended) import . "fmt" // Now can use Println directly

Creating Your Own Package

1. Create a directory for your package (e.g., mypackage)

2. Create a file (e.g., mypackage.go) with package declaration:

package mypackage // Exported function (capitalized) func Add(a, b int) int { return a + b } // Private function (lowercase) func internalFunc() { // ... }

3. In your main program, import and use the package:

package main import ( "fmt" "./mypackage" // Import local package ) func main() { result := mypackage.Add(5, 3) fmt.Println(result) // 8 }

Common Standard Library Packages

  • fmt: Formatted I/O
  • io: Basic I/O interfaces
  • os: Operating system functions
  • strconv: String conversions
  • math: Mathematical functions
  • net/http: HTTP server and client
  • encoding/json: JSON encoding/decoding
  • time: Time measurement and display

Data Structures

Go has several built-in data structures for organizing data efficiently.

Arrays and Slices

// Arrays (fixed size) var numbers [5]int // Declare array numbers = [5]int{1, 2, 3} // Initialize (remaining are zero) primes := [3]int{2, 3, 5} // Shorthand declaration // Slices (dynamic arrays) var slice []int // Declare slice slice = []int{1, 2, 3} // Initialize slice = make([]int, 5) // Using make (length 5, capacity 5) slice = make([]int, 3, 5) // Length 3, capacity 5 // Slice operations slice = append(slice, 4) // Append element length := len(slice) // Get length capacity := cap(slice) // Get capacity // Slicing subSlice := slice[1:3] // Elements 1 to 2 (exclusive)

Maps

// Map declaration and initialization var person map[string]interface{} // Key: string, Value: any type person = make(map[string]interface{}) // Shorthand declaration with initialization person := map[string]interface{}{ "name": "Alice", "age": 25, } // Map operations person["occupation"] = "Engineer" // Add/update age := person["age"] // Retrieve value delete(person, "age") // Delete key-value pair // Check if key exists if value, exists := person["age"]; exists { fmt.Println("Age:", value) }

Structs

// Define a struct type type Person struct { Name string Age int City string } // Create struct instances var p1 Person p1 = Person{"Alice", 25, "New York"} // Positional fields p2 := Person{ // Named fields Name: "Bob", Age: 30, City: "London", } // Access fields fmt.Println(p1.Name) // Alice p2.Age = 31 // Update field

To-Do List Example

package main import ( "fmt" "os" ) type Task struct { description string completed bool } func main() { var tasks []Task for { fmt.Println("\nTo-Do List:") for i, task := range tasks { status := " " if task.completed { status = "‚úì" } fmt.Printf("%d. [%s] %s\n", i+1, status, task.description) } fmt.Print("\nOptions: (a)dd, (t)oggle, (q)uit: ") var choice string fmt.Scanln(&choice) switch choice { case "a": fmt.Print("Enter task description: ") var desc string fmt.Scanln(&desc) tasks = append(tasks, Task{description: desc}) case "t": fmt.Print("Enter task number to toggle: ") var num int fmt.Scanln(&num) if num > 0 && num <= len(tasks) { tasks[num-1].completed = !tasks[num-1].completed } case "q": os.Exit(0) default: fmt.Println("Invalid option") } } }

This simple program demonstrates structs, slices, and basic control flow in a practical example.

Pointers

Go has pointers but no pointer arithmetic, making them safer than in languages like C.

// Pointer basics var x int = 10 var p *int = &x // p points to x fmt.Println(*p) // Dereference pointer (10) *p = 20 // Change value through pointer fmt.Println(x) // Now 20 // Pointer to struct person := Person{Name: "Alice"} pp := &person pp.Age = 25 // Same as (*pp).Age = 25 fmt.Println(person) // New function (allocates memory) ptr := new(int) // *int type, initialized to 0 *ptr = 100

When to Use Pointers

  • When you need to modify the original variable in a function
  • When working with large structs to avoid copying
  • When you need to represent optional values (nil pointer)
// Function that modifies original value func increment(x *int) { *x++ } func main() { num := 5 increment(&num) fmt.Println(num) // 6 }
Unlike C, Go has garbage collection, so you don't need to manually free memory allocated with new.

Methods and Interfaces

Go doesn't have classes but supports methods on types and interface-based polymorphism.

Methods

// Define method on a type type Rectangle struct { width, height float64 } // Method with value receiver func (r Rectangle) Area() float64 { return r.width * r.height } // Method with pointer receiver func (r *Rectangle) Scale(factor float64) { r.width *= factor r.height *= factor } func main() { rect := Rectangle{width: 10, height: 5} fmt.Println(rect.Area()) // 50 rect.Scale(2) fmt.Println(rect.Area()) // 200 }

Interfaces

// Define an interface type Shape interface { Area() float64 Perimeter() float64 } // Rectangle implements Shape implicitly func (r Rectangle) Perimeter() float64 { return 2 * (r.width + r.height) } // Another shape implementing Shape type Circle struct { radius float64 } func (c Circle) Area() float64 { return math.Pi * c.radius * c.radius } func (c Circle) Perimeter() float64 { return 2 * math.Pi * c.radius } // Function accepting any Shape func printShapeInfo(s Shape) { fmt.Printf("Area: %.2f, Perimeter: %.2f\n", s.Area(), s.Perimeter()) } func main() { shapes := []Shape{ Rectangle{width: 10, height: 5}, Circle{radius: 7}, } for _, shape := range shapes { printShapeInfo(shape) } }
In Go, interfaces are satisfied implicitly. A type implements an interface by implementing its methods - no explicit declaration is needed.

Concurrency in Go

Go has excellent built-in support for concurrent programming with goroutines and channels.

Goroutines

// Simple goroutine func sayHello() { fmt.Println("Hello from goroutine") } func main() { go sayHello() // Start a new goroutine time.Sleep(100 * time.Millisecond) // Wait for goroutine to complete } // Goroutine with anonymous function go func(msg string) { fmt.Println(msg) }("Hello")

Channels

// Basic channel operations messages := make(chan string) // Create channel // Send to channel (in goroutine) go func() { messages <- "Hello" // Send message }() // Receive from channel msg := <-messages fmt.Println(msg) // Hello // Buffered channels messages := make(chan string, 2) // Buffer size 2 messages <- "First" messages <- "Second" fmt.Println(<-messages) // First fmt.Println(<-messages) // Second // Channel synchronization done := make(chan bool) go func() { fmt.Println("Working...") time.Sleep(time.Second) done <- true }() <-done // Wait for goroutine to finish fmt.Println("Done")

Select Statement

// Select allows waiting on multiple channels func main() { c1 := make(chan string) c2 := make(chan string) go func() { time.Sleep(1 * time.Second) c1 <- "one" }() go func() { time.Sleep(2 * time.Second) c2 <- "two" }() for i := 0; i < 2; i++ { select { case msg1 := <-c1: fmt.Println("Received", msg1) case msg2 := <-c2: fmt.Println("Received", msg2) } } }
Goroutines are lightweight, but you should still be careful about creating too many. Use worker pools or other patterns for high-concurrency scenarios.

Error Handling

Go uses explicit error returns rather than exceptions. Functions that can fail typically return an error as their last return value.

// Basic error handling f, err := os.Open("filename.txt") if err != nil { log.Fatal(err) // Print error and exit } defer f.Close() // Ensure file is closed // Custom errors func divide(a, b float64) (float64, error) { if b == 0.0 { return 0.0, fmt.Errorf("cannot divide by zero") } return a / b, nil } // Using errors result, err := divide(10.0, 0.0) if err != nil { fmt.Println("Error:", err) } else { fmt.Println("Result:", result) } // Error wrapping if err != nil { return fmt.Errorf("operation failed: %w", err) } // Checking error types var pathError *os.PathError if errors.As(err, &pathError) { fmt.Println("Failed at path:", pathError.Path) }

Defer, Panic, and Recover

// Defer executes when function exits func readFile() { f, err := os.Open("file.txt") if err != nil { log.Fatal(err) } defer f.Close() // Will be called when readFile returns // Process file } // Panic stops normal execution func mayPanic() { panic("a problem occurred") } // Recover can catch panics func safeCall() { defer func() { if r := recover(); r != nil { fmt.Println("Recovered:", r) } }() mayPanic() fmt.Println("This won't run") } func main() { safeCall() fmt.Println("Program continues") }
In Go, it's idiomatic to handle errors immediately where they occur rather than throwing exceptions up the call stack.

File Handling

Go's standard library provides comprehensive support for file operations.

// Reading files data, err := os.ReadFile("file.txt") // Read entire file if err != nil { log.Fatal(err) } fmt.Println(string(data)) // Writing files content := []byte("Hello, Go!") err := os.WriteFile("output.txt", content, 0644) if err != nil { log.Fatal(err) } // Working with file objects file, err := os.Create("data.txt") if err != nil { log.Fatal(err) } defer file.Close() writer := bufio.NewWriter(file) fmt.Fprintln(writer, "Line 1") fmt.Fprintln(writer, "Line 2") writer.Flush() // Ensure data is written // Reading line by line file, err := os.Open("data.txt") if err != nil { log.Fatal(err) } defer file.Close() scanner := bufio.NewScanner(file) for scanner.Scan() { fmt.Println(scanner.Text()) } if err := scanner.Err(); err != nil { log.Fatal(err) }

Working with JSON

// Marshaling (struct to JSON) type Person struct { Name string `json:"name"` Age int `json:"age"` } person := Person{"Alice", 25} jsonData, err := json.Marshal(person) if err != nil { log.Fatal(err) } fmt.Println(string(jsonData)) // {"name":"Alice","age":25} // Unmarshaling (JSON to struct) var decodedPerson Person err = json.Unmarshal(jsonData, &decodedPerson) if err != nil { log.Fatal(err) } fmt.Printf("%+v\n", decodedPerson) // {Name:Alice Age:25}
