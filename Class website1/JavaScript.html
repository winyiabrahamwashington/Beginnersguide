JavaScript Fundamentals

What is JavaScript?

JavaScript is a high-level, interpreted programming language that conforms to the ECMAScript specification. It is a language that is also characterized as dynamic, weakly typed, prototype-based and multi-paradigm.

JavaScript is the language of the web

JavaScript is one of the core technologies of the World Wide Web, alongside HTML and CSS. Over 97% of websites use JavaScript on the client side for web page behavior, often incorporating third-party libraries.

Where JavaScript is Used:

  • Web development (front-end and back-end with Node.js)
  • Mobile app development (React Native, Ionic)
  • Game development
  • Desktop applications (Electron)
  • Server-side applications

Key Features of JavaScript:

  • Client-side execution in web browsers
  • Event-driven programming model
  • Asynchronous programming with callbacks and promises
  • Object-oriented but prototype-based
  • First-class functions (functions are treated as objects)
  • Dynamic typing (variables can hold any type of data)

Variables and Data Types

Variables are containers for storing data values. In JavaScript, you can declare variables using var, let, or const.

// Variable declarations
let message = "Hello";  // string
const PI = 3.14;        // number (constant)
var isActive = true;    // boolean
let age = null;         // null
let name;               // undefined

// Modern JavaScript prefers let and const over var
let count = 10;
const MAX_SIZE = 100;

JavaScript has dynamic types. This means the same variable can be used to hold different data types:

let x;           // x is undefined
x = 5;           // x is a number
x = "John";      // x is a string
x = true;        // x is a boolean

JavaScript Data Types:

  • Primitive types: String, Number, Boolean, Null, Undefined, Symbol (ES6), BigInt (ES2020)
  • Object types: Object, Array, Date, Function, etc.

Variables declared with const cannot be reassigned, while let variables can be. var has function scope, while let and const have block scope.

Variable Best Practices:

  • Use const by default, let when you need to reassign, and avoid var
  • Use meaningful variable names (e.g., userAge instead of ua)
  • Initialize variables when you declare them
  • Use camelCase for variable names (e.g., firstName)
  • Constants should be in UPPER_CASE (e.g., MAX_USERS)

Functions and Control Flow

Functions are blocks of code designed to perform a particular task. They are executed when "called" or "invoked".

// Function declaration
function greet(name) {
    return "Hello, " + name + "!";
}

// Function expression (assigned to a variable)
const greet = function(name) {
    return `Hello, ${name}!`;
}

// Arrow function (ES6)
const greet = (name) => `Hello, ${name}!`;

// Calling functions
console.log(greet("Alice")); // "Hello, Alice!"

Control flow refers to the order in which statements are executed in a program. JavaScript includes conditional statements and loops to control the flow of execution.

// If-else statement
let age = 18;
if (age < 18) {
    console.log("You're a minor");
} else if (age >= 18 && age < 65) {
    console.log("You're an adult");
} else {
    console.log("You're a senior");
}

// Switch statement
let day = "Monday";
switch(day) {
    case "Monday":
        console.log("Start of work week");
        break;
    case "Friday":
        console.log("Weekend is coming!");
        break;
    default:
        console.log("It's a regular day");
}

// For loop
for (let i = 0; i < 5; i++) {
    console.log(i); // 0, 1, 2, 3, 4
}

// While loop
let i = 0;
while (i < 5) {
    console.log(i);
    i++;
}

Common Uses of Functions:

  • Code reuse - write once, use many times
  • Organizing code into logical blocks
  • Event handling (click handlers, etc.)
  • Asynchronous operations (callbacks, promises)

Function Best Practices:

  • Functions should do one thing and do it well
  • Keep functions small and focused
  • Use descriptive names (verbs for actions, nouns for data retrieval)
  • Avoid too many parameters (3 or fewer is ideal)
  • Use arrow functions for callbacks and when you need lexical this

DOM Manipulation

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content.

DOM is the interface between JavaScript and HTML/CSS

JavaScript can manipulate the DOM to dynamically change the content and appearance of a web page.

// Selecting elements
const heading = document.getElementById("main-heading");
const buttons = document.querySelectorAll(".btn");

// Changing content
heading.textContent = "New Heading";
heading.innerHTML = "<em>New</em> Heading";

// Changing styles
heading.style.color = "blue";
heading.style.fontSize = "2rem";

// Adding/removing classes
heading.classList.add("highlight");
heading.classList.remove("old-class");

// Creating new elements
const newParagraph = document.createElement("p");
newParagraph.textContent = "This is a new paragraph";
document.body.appendChild(newParagraph);

// Event handling
const button = document.querySelector("#myButton");
button.addEventListener("click", function() {
    alert("Button was clicked!");
});

Common DOM Manipulation Tasks:

  • Form validation
  • Creating interactive elements (dropdowns, modals)
  • Updating content without page reload
  • Animating elements
  • Fetching and displaying data from APIs

DOM Best Practices:

  • Cache DOM references in variables if used multiple times
  • Use textContent instead of innerHTML when possible for security
  • Batch DOM updates to minimize reflows and repaints
  • Use event delegation for dynamic elements
  • Consider using modern frameworks (React, Vue, Angular) for complex UIs

Event Handling

Events are actions or occurrences that happen in the system you are programming, which the system tells you about so you can respond to them in some way if desired.

// Common events to handle:
// click, mouseover, mouseout, keydown, keyup, submit, change, load

// Adding event listeners
const button = document.querySelector("#myButton");

// Method 1: Using anonymous function
button.addEventListener("click", function() {
    console.log("Button was clicked!");
});

// Method 2: Using named function
function handleClick() {
    console.log("Button was clicked!");
}
button.addEventListener("click", handleClick);

// Method 3: Inline HTML event handler (not recommended)
<button onclick="handleClick()">Click me</button>

// Event object
button.addEventListener("click", function(event) {
    console.log(event); // Information about the event
    console.log(event.target); // The element that triggered the event
});

// Preventing default behavior
const form = document.querySelector("#myForm");
form.addEventListener("submit", function(event) {
    event.preventDefault(); // Stop form from submitting
    // Handle form data manually
});

Common Event Handling Scenarios:

  • Button clicks
  • Form submissions
  • Keyboard interactions
  • Mouse movements
  • Window events (resize, scroll)
  • Touch events for mobile devices

Event delegation is a technique where you delegate events to parent elements rather than setting them on each child element. This is useful for dynamically added elements or when you have many similar elements.

// Instead of adding listeners to each button
document.querySelector("#buttonContainer").addEventListener("click", function(event) {
    if (event.target.classList.contains("btn")) {
        console.log("Button clicked:", event.target.textContent);
    }
});

Event Handling Best Practices:

  • Use event delegation for dynamic content
  • Remove event listeners when no longer needed to prevent memory leaks
  • Be mindful of event propagation (bubbling and capturing)
  • Use passive event listeners for scroll events to improve performance
  • Consider throttling or debouncing frequent events like scroll or resize

Arrays and Objects

Arrays are used to store multiple values in a single variable. Objects are collections of key-value pairs.

// Arrays
let fruits = ["Apple", "Banana", "Orange"];
console.log(fruits[0]); // "Apple"

// Array methods
fruits.push("Mango"); // Add to end
fruits.pop(); // Remove from end
fruits.unshift("Strawberry"); // Add to beginning
fruits.shift(); // Remove from beginning

// Iterating over arrays
fruits.forEach(function(fruit) {
    console.log(fruit);
});

// Objects
let person = {
    firstName: "John",
    lastName: "Doe",
    age: 30,
    hobbies: ["reading", "coding"],
    address: {
        street: "123 Main St",
        city: "Boston"
    }
};

// Accessing object properties
console.log(person.firstName); // "John"
console.log(person["lastName"]); // "Doe"
console.log(person.hobbies[0]); // "reading"

// Adding/modifying properties
person.email = "john@example.com";
person.age = 31;

// Object methods
person.fullName = function() {
    return this.firstName + " " + this.lastName;
};

Common Array and Object Operations:

  • Storing and organizing data
  • Processing lists of items (map, filter, reduce)
  • Working with JSON data from APIs
  • Creating complex data structures

Modern JavaScript Features:

  • Destructuring: Extract values from arrays/objects into variables
  • Spread/Rest: Expand or collect elements with ... syntax
  • Template literals: Use backticks for string interpolation
  • Arrow functions: Concise syntax with lexical this
  • Modules: Organize code into reusable modules with import/export
  • Promises/Async-Await: Handle asynchronous operations cleanly
