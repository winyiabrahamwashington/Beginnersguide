Getting Started with TypeScript

TypeScript is a strongly typed superset of JavaScript that compiles to plain JavaScript. Developed by Microsoft, it adds optional static typing and class-based object-oriented programming to JavaScript.

TypeScript is widely used in large-scale JavaScript applications

TypeScript is not a completely different language - it's JavaScript with additional features. All valid JavaScript code is also valid TypeScript code.

What You'll Need

  • Node.js installed on your computer
  • A text editor or IDE (VS Code is highly recommended)
  • Basic knowledge of JavaScript

Why Learn TypeScript?

TypeScript offers several advantages over plain JavaScript:

  • Static Typing: Catch errors during development rather than runtime
  • Better Tooling: Improved autocompletion, navigation, and refactoring
  • Enhanced Readability: Explicit types make code more self-documenting
  • Growing Popularity: Used by Angular, Vue 3, React (optionally), and many large companies
  • Future JavaScript Features: Use modern features that compile to older JavaScript versions

Your First TypeScript Program

Here's the classic "Hello, World!" program in TypeScript:

// hello.ts function greet(name: string): string { return `Hello, ${name}!`; } const message: string = greet("TypeScript"); console.log(message);

To compile and run this:

  1. Save it as hello.ts
  2. Run tsc hello.ts to compile to JavaScript
  3. Run node hello.js to execute the compiled code

Basic Types

TypeScript provides several basic types that you can use to annotate your variables and function parameters.

// Primitive types let isDone: boolean = false; let count: number = 10; let name: string = "Alice"; // Arrays let numbers: number[] = [1, 2, 3]; let names: Array = ["Alice", "Bob"]; // Generic array type // Tuples - fixed length arrays where each element has its own type let tuple: [string, number] = ["Alice", 25]; // Enums - way to give friendly names to sets of numeric values enum Color {Red, Green, Blue} let c: Color = Color.Green; // Any - opt-out of type checking let notSure: any = 4; notSure = "maybe a string instead"; // Void - absence of any type (usually for functions that don't return) function warnUser(): void { console.log("This is a warning"); } // Never - represents values that never occur function error(message: string): never { throw new Error(message); }

Type Inference

TypeScript can often infer types without explicit annotations:

let x = 3; // TypeScript infers x is number let y = "hello"; // TypeScript infers y is string // Contextual typing window.onmousedown = function(event) { console.log(event.button); // TypeScript knows event is MouseEvent };
While TypeScript can infer types, it's often good practice to explicitly annotate function return types and complex object shapes for better documentation and error checking.

Interfaces

Interfaces are a powerful way to define contracts within your code and with code outside of your project.

// Basic interface interface Person { firstName: string; lastName: string; age?: number; // Optional property readonly ssn: string; // Readonly property } function greet(person: Person): string { return `Hello, ${person.firstName} ${person.lastName}`; } let user = {firstName: "John", lastName: "Doe", ssn: "123-45-6789"}; console.log(greet(user)); // Interface with function type interface SearchFunc { (source: string, subString: string): boolean; } let mySearch: SearchFunc; mySearch = function(src: string, sub: string) { return src.search(sub) > -1; } // Interface for array type interface StringArray { [index: number]: string; } let myArray: StringArray; myArray = ["Bob", "Fred"];

Extending Interfaces

interface Shape { color: string; } interface Square extends Shape { sideLength: number; } let square = {} as Square; square.color = "blue"; square.sideLength = 10; // Multiple interface inheritance interface PenStroke { penWidth: number; } interface SquareWithPen extends Shape, PenStroke { sideLength: number; }
Interfaces only exist at compile time - they are removed when TypeScript is compiled to JavaScript. They are purely for type checking during development.

Classes

TypeScript classes add type annotations and visibility modifiers to JavaScript classes.

class Animal { // Properties private name: string; protected age: number; public readonly species: string; // Constructor constructor(name: string, age: number, species: string) { this.name = name; this.age = age; this.species = species; } // Method move(distance: number = 0) { console.log(`${this.name} moved ${distance}m.`); } } class Dog extends Animal { // Additional property breed: string; constructor(name: string, age: number, breed: string) { super(name, age, "Canine"); this.breed = breed; } // Override method move(distance = 5) { console.log("Running..."); super.move(distance); } // New method bark() { console.log("Woof! Woof!"); } } const dog = new Dog("Buddy", 3, "Golden Retriever"); dog.move(); dog.bark();

Access Modifiers

Modifier Description
public Default. Accessible anywhere
private Only accessible within class
protected Accessible within class and subclasses
readonly Can only be set during initialization

Abstract Classes

abstract class Department { constructor(public name: string) {} abstract printMeeting(): void; // Must be implemented in derived classes printDepartmentName(): void { console.log(`Department: ${this.name}`); } } class AccountingDepartment extends Department { constructor() { super("Accounting"); } printMeeting(): void { console.log("Accounting meeting at 10AM"); } } let department: Department; // Can create a reference to abstract type department = new AccountingDepartment(); // Can assign subclass instance department.printMeeting();

Functions

TypeScript adds type annotations to JavaScript functions, enabling better parameter and return type checking.

// Function with typed parameters and return value function add(x: number, y: number): number { return x + y; } // Optional parameters function buildName(firstName: string, lastName?: string): string { return lastName ? `${firstName} ${lastName}` : firstName; } // Default parameters function createUser(name: string, isAdmin: boolean = false) { return { name, isAdmin }; } // Rest parameters function sumAll(...numbers: number[]): number { return numbers.reduce((total, num) => total + num, 0); } // Function type let myAdd: (x: number, y: number) => number; myAdd = function(x: number, y: number): number { return x + y; }; // Arrow functions const multiply = (x: number, y: number): number => x * y;

Function Overloads

// Overload signatures function makeDate(timestamp: number): Date; function makeDate(year: number, month: number, day: number): Date; // Implementation function makeDate(timestampOrYear: number, month?: number, day?: number): Date { if (month !== undefined && day !== undefined) { return new Date(timestampOrYear, month, day); } else { return new Date(timestampOrYear); } } const d1 = makeDate(12345678); const d2 = makeDate(2023, 5, 15); // const d3 = makeDate(2023, 5); // Error: No overload expects 2 arguments
Function overloads in TypeScript don't create multiple functions like in some other languages. They simply provide multiple ways to call a single function with different parameter combinations.

Generics

Generics provide a way to create reusable components that work with multiple types rather than a single one.

// Generic function function identity(arg: T): T { return arg; } let output1 = identity("myString"); let output2 = identity(100); let output3 = identity("myString"); // Type inference // Generic interface interface GenericIdentityFn { (arg: T): T; } let myIdentity: GenericIdentityFn = identity; // Generic class class GenericNumber { zeroValue: T; add: (x: T, y: T) => T; } let myGenericNumber = new GenericNumber(); myGenericNumber.zeroValue = 0; myGenericNumber.add = function(x, y) { return x + y; }; // Generic constraints interface Lengthwise { length: number; } function loggingIdentity(arg: T): T { console.log(arg.length); return arg; } loggingIdentity("hello"); // OK loggingIdentity({length: 10, value: 3}); // OK // loggingIdentity(3); // Error: number doesn't have .length

Using Type Parameters in Generic Constraints

function getProperty(obj: T, key: K) { return obj[key]; } let x = { a: 1, b: 2, c: 3 }; getProperty(x, "a"); // OK // getProperty(x, "d"); // Error: Argument of type '"d"' is not assignable
While generics are powerful, overusing them can make your code harder to understand. Use them when you need type flexibility but want to maintain type safety.

Advanced Types

TypeScript offers several advanced type features that help model complex scenarios.

Union and Intersection Types

// Union types function padLeft(value: string, padding: string | number) { if (typeof padding === "number") { return Array(padding + 1).join(" ") + value; } return padding + value; } // Intersection types interface BusinessPartner { name: string; credit: number; } interface Contact { email: string; phone: string; } type Customer = BusinessPartner & Contact; let customer: Customer = { name: "Acme Corp", credit: 1000000, email: "contact@acme.com", phone: "555-1234" };

Type Guards and Differentiating Types

// Using typeof function printAll(strs: string | string[] | null) { if (typeof strs === "object") { for (const s of strs!) { // Non-null assertion console.log(s); } } else if (typeof strs === "string") { console.log(strs); } } // Using instanceof class Bird { fly() { console.log("flying"); } } class Fish { swim() { console.log("swimming"); } } function move(pet: Bird | Fish) { if (pet instanceof Bird) { pet.fly(); } else { pet.swim(); } } // User-defined type guards function isFish(pet: Fish | Bird): pet is Fish { return (pet as Fish).swim !== undefined; } function getSmallPet(): Fish | Bird { return Math.random() > 0.5 ? new Fish() : new Bird(); } let pet = getSmallPet(); if (isFish(pet)) { pet.swim(); } else { pet.fly(); }

Mapped Types

interface Person { name: string; age: number; location: string; } type ReadonlyPerson = Readonly; type PartialPerson = Partial; type PickNameAndAge = Pick; type OmitAge = Omit; // Custom mapped type type Nullable = { [P in keyof T]: T[P] | null; }; type NullablePerson = Nullable;

Modules

TypeScript supports modern JavaScript modules and adds type safety to imports and exports.

// math.ts - Exporting export function square(x: number): number { return x * x; } export const PI = 3.14159; export interface Circle { radius: number; area: () => number; } // Default export export default class Calculator { add(x: number, y: number): number { return x + y; } } // app.ts - Importing import Calculator, { square, PI, Circle } from "./math"; const calc = new Calculator(); console.log(calc.add(2, 3)); console.log(square(5)); const circle: Circle = { radius: 10, area: () => PI * 10 * 10 };

Namespaces

namespace Validation { export interface StringValidator { isAcceptable(s: string): boolean; } const lettersRegexp = /^[A-Za-z]+$/; const numberRegexp = /^[0-9]+$/; export class LettersOnlyValidator implements StringValidator { isAcceptable(s: string) { return lettersRegexp.test(s); } } export class ZipCodeValidator implements StringValidator { isAcceptable(s: string) { return s.length === 5 && numberRegexp.test(s); } } } // Usage let validators: { [s: string]: Validation.StringValidator } = {}; validators["Letters only"] = new Validation.LettersOnlyValidator(); validators["Zip code"] = new Validation.ZipCodeValidator();
While namespaces are still supported in TypeScript, ES6 modules are generally preferred for new projects as they are the standard in modern JavaScript.

Decorators

Decorators provide a way to add annotations and a meta-programming syntax for class declarations and members.

// Class decorator function sealed(constructor: Function) { Object.seal(constructor); Object.seal(constructor.prototype); } @sealed class Greeter { greeting: string; constructor(message: string) { this.greeting = message; } greet() { return "Hello, " + this.greeting; } } // Method decorator function enumerable(value: boolean) { return function (target: any, propertyKey: string, descriptor: PropertyDescriptor) { descriptor.enumerable = value; }; } class Person { name: string; constructor(name: string) { this.name = name; } @enumerable(false) greet() { return "Hello, " + this.name; } } // Property decorator function format(formatString: string) { return function (target: any, propertyKey: string): any { let value = target[propertyKey]; function getter() { return `${formatString} ${value}`; } function setter(newVal: string) { value = newVal; } return { get: getter, set: setter, enumerable: true, configurable: true }; } } class Greeter2 { @format("Hello") greeting: string; constructor(message: string) { this.greeting = message; } } let greeter2 = new Greeter2("World"); console.log(greeter2.greeting); // "Hello World"
Decorators are an experimental feature in TypeScript. To use them, you must enable the experimentalDecorators compiler option in your tsconfig.json file.
