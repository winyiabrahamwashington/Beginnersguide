Getting Started with HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure of a web page and consists of a series of elements that tell browsers how to display content.

HTML is used to structure content on web pages

HTML is not a programming language‚Äîit's a markup language that defines the structure of your content.

What You'll Need

  • A text editor (VS Code, Sublime Text, or even Notepad)
  • A web browser (Chrome, Firefox, Edge)
  • Basic computer skills

What is HTML Used For?

HTML is used to create the structure and content of web pages. It's the skeleton that holds everything together:

  • Defining headings, paragraphs, and text
  • Creating links between pages
  • Embedding images and videos
  • Building forms for user input
  • Organizing content with lists and tables

How HTML Works

When you visit a website, your browser:

  1. Requests the HTML file from the server
  2. Reads the HTML code
  3. Renders (displays) the page based on the HTML instructions

HTML Example

Here's a very simple HTML document that displays "Hello, World!":

<!DOCTYPE html> <html> <head> <title>My First Page</title> </head> <body> <h1>Hello, World!</h1> </body> </html>

Basic Structure of an HTML Document

Every HTML document has a basic structure that must be followed. Here's the minimal code needed for a valid HTML page:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Document Title</title> </head> <body> <!-- Your content goes here --> </body> </html>

Key Components Explained

  • <!DOCTYPE html> - Declares this is an HTML5 document (must be first line)
  • <html> - The root element wrapping all content
  • <head> - Contains meta information not displayed on page
  • <meta charset="UTF-8"> - Defines character encoding
  • <meta name="viewport"> - Ensures proper mobile rendering
  • <title> - Sets the browser tab title
  • <body> - Contains all visible page content

Understanding HTML Syntax

HTML uses a simple syntax made up of elements:

  • Tags: Keywords surrounded by angle brackets (< >)
  • Elements: Opening tag + content + closing tag
  • Attributes: Additional information about elements (inside opening tag)
Component Description Example
Opening Tag Starts an HTML element <p>
Content The actual content to display This is a paragraph
Closing Tag Ends an HTML element </p>
Element The complete tag set <p>This is a paragraph</p>

HTML Tags and Elements

HTML uses "tags" to create elements that structure content. Most elements have an opening tag, content, and closing tag:

<tagname>Content goes here...</tagname>

Anatomy of an HTML element

Common HTML Tags

Headings

HTML provides six levels of headings from <h1> (most important) to <h6> (least important):

<h1>Main Heading</h1> <h2>Subheading</h2> <h3>Sub-subheading</h3>

Paragraphs

The <p> tag defines a paragraph. Browsers automatically add spacing before and after:

<p>This is a paragraph.</p> <p>This is another paragraph.</p>

Lists

Create ordered (numbered) and unordered (bulleted) lists:

<!-- Unordered list --> <ul> <li>First item</li> <li>Second item</li> </ul> <!-- Ordered list --> <ol> <li>First step</li> <li>Second step</li> </ol>

Self-Closing Tags

Some HTML elements don't have closing tags because they don't contain content. These are called void elements:

<!-- Line break --> <br> <!-- Image --> <img src="image.jpg" alt="Description"> <!-- Input field --> <input type="text">
In HTML5, you don't need to include the slash in self-closing tags (like <br/>), though it's still valid if you prefer that style.

HTML Attributes

Attributes provide additional information about HTML elements. They are always specified in the start tag and usually come in name/value pairs like name="value".

<a href="https://example.com" target="_blank">Visit Example</a> <img src="image.jpg" alt="Description" width="500"> <input type="text" id="username" name="username">

Common Attributes

  • class - Specifies one or more class names for an element (used for CSS and JavaScript)
  • id - Specifies a unique id for an element
  • style - Specifies an inline CSS style for an element
  • title - Specifies extra information about an element (shown as tooltip)
  • data-* - Used to store custom data private to the page or application
Always include descriptive alt text for images for accessibility. Use empty alt (alt="") for decorative images.

Boolean Attributes

Some attributes don't need a value - their presence alone makes them true:

<input type="checkbox" checked> <button disabled>Can't Click Me</button>

These are called boolean attributes. You can also write them with values (e.g., checked="checked"), but it's not necessary in HTML5.

Basic Formatting Tags

HTML provides various tags for text formatting and styling:

<b> - Bold text <strong> - Important text (semantic) <i> - Italic text <em> - Emphasized text (semantic) <mark> - Marked/highlighted text <small> - Smaller text <del> - Deleted text <ins> - Inserted text <sub> - Subscript text <sup> - Superscript text
While <b> and <i> work, it's better to use <strong> and <em> for semantic meaning and accessibility.

Example of Formatting

<p>This is <b>bold</b> and this is <strong>strong</strong>.</p> <p>This is <i>italic</i> and this is <em>emphasized</em>.</p> <p>Don't forget to <mark>highlight</mark> important text.</p> <p>This is <small>smaller text</small>.</p> <p>My favorite color is <del>blue</del> <ins>red</ins>.</p> <p>H<sub>2</sub>O is water.</p> <p>E = mc<sup>2</sup> is Einstein's equation.</p>

HTML Comments

You can add comments to your HTML that won't be displayed in the browser:

<!-- This is a comment --> <!-- This is a multi-line comment in HTML -->

Comments are useful for:

  • Explaining your code
  • Temporarily disabling code
  • Leaving notes for other developers

Links and Navigation

Hyperlinks are what makes the web "web-like". The <a> tag defines a hyperlink, which is used to link from one page to another.

Creating Links

The <a> tag creates hyperlinks. The href attribute specifies the destination:

<!-- External link --> <a href="https://www.example.com">Visit Example</a> <!-- Internal link --> <a href="about.html">About Us</a> <!-- Link that opens in new tab --> <a href="https://www.example.com" target="_blank" rel="noopener">Open in new tab</a> <!-- Email link --> <a href="mailto:someone@example.com">Send Email</a> <!-- Phone link --> <a href="tel:+1234567890">Call Us</a>

Navigation Menu Example

<nav> <ul> <li><a href="index.html">Home</a></li> <li><a href="about.html">About</a></li> <li><a href="services.html">Services</a></li> <li><a href="contact.html">Contact</a></li> </ul> </nav>

An image displayed using the HTML <img> tag

Linking to Page Sections

You can link to specific sections within a page using IDs:

<!-- First, add an id to an element --> <h2 id="section1">Section 1</h2> <!-- Then link to it --> <a href="#section1">Jump to Section 1</a>

Images and Media

Images and media make websites more engaging and interactive. HTML provides tags to embed these elements.

Adding Images

The <img> tag embeds images. It's self-closing and requires src and alt attributes:

<img src="image.jpg" alt="Description of image"> <img src="logo.png" alt="Company logo" width="200" height="100">

Adding Video and Audio

<!-- HTML5 Video --> <video width="320" height="240" controls> <source src="movie.mp4" type="video/mp4"> Your browser does not support the video tag. </video> <!-- HTML5 Audio --> <audio controls> <source src="audio.mp3" type="audio/mpeg"> Your browser does not support the audio element. </audio>
Always include fallback content between the opening and closing tags for <video> and <audio> for browsers that don't support these elements.

Image Formats

Common image formats for the web:

Format Best For Extension

Forms and User Input

Forms allow users to enter data that can be sent to a server for processing. They are essential for user interaction.

Basic Form Structure

<form action="/submit" method="post"> <label for="name">Name:</label> <input type="text" id="name" name="name" required> <label for="email">Email:</label> <input type="email" id="email" name="email" required> <label for="message">Message:</label> <textarea id="message" name="message" rows="4"></textarea> <button type="submit">Send</button> </form>

Common Form Elements

  • <input> - For text, email, password, radio buttons, checkboxes, etc.
  • <textarea> - For multi-line text input
  • <select> - For dropdown menus
  • <button> - For submit/reset buttons
  • <label> - To properly label form controls
  • <fieldset> - Groups related elements in a form
  • <legend> - Defines a caption for a <fieldset>

A simple HTML form with various input types

Common Input Types

HTML5 introduced many new input types that provide better controls and validation:

<input type="text"> <!-- Basic text input --> <input type="password"> <!-- Password field (hidden) --> <input type="email"> <!-- Email with validation --> <input type="number"> <!-- Number input --> <input type="date"> <!-- Date picker --> <input type="color"> <!-- Color picker --> <input type="range"> <!-- Slider control --> <input type="checkbox"> <!-- Checkbox --> <input type="radio"> <!-- Radio button --> <input type="file"> <!-- File upload --> <input type="submit"> <!-- Submit button -->

Semantic HTML5

Semantic HTML introduces elements that clearly describe their meaning to both browsers and developers. This improves accessibility, SEO, and maintainability.

<header> <h1>Website Title</h1> <nav> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> </ul> </nav> </header> <main> <article> <h2>Article Title</h2> <p>Article content...</p> </article> <aside> <h3>Related Links</h3> <ul> <li><a href="#">Link 1</a></li> </ul> </aside> </main> <footer> <p>Copyright © 2023</p> </footer>

How semantic elements structure a web page

Key Semantic Elements

  • <header> - Introductory content
  • <nav> - Navigation links
  • <main> - Primary content
  • <article> - Self-contained composition
  • <section> - Thematic grouping
  • <aside> - Indirectly related content
  • <footer> - Footer content
  • <figure> - Self-contained content like illustrations
  • <figcaption> - Caption for a <figure>
  • <time> - Machine-readable date/time
Using semantic HTML improves accessibility, SEO, and makes your code easier to understand and maintain.

Why Semantic HTML Matters

Semantic HTML provides important benefits:

  1. Accessibility: Screen readers can better interpret page structure
  2. SEO: Search engines understand content better
  3. Maintainability: Code is easier to read and update
  4. Future Compatibility: New devices and browsers will handle semantic elements properly

Debugging and Testing

Validating and testing your HTML ensures it works correctly across different browsers and devices.

HTML Validation

Use the W3C Validator to check your HTML for errors:

https://validator.w3.org/

Browser Developer Tools

All modern browsers include developer tools (F12 or right-click ‚Üí Inspect) that allow you to:

  • Inspect HTML elements
  • Debug JavaScript
  • Test responsive designs
  • Analyze performance

Cross-Browser Testing

Test your website in different browsers (Chrome, Firefox, Safari, Edge) to ensure compatibility.

Always test your website on actual mobile devices, not just browser emulators, for accurate results.

Common HTML Mistakes to Avoid

  • Missing closing tags: Always close your HTML elements properly
  • Nesting errors: Don't overlap elements incorrectly (e.g., <p><div></p></div>)
  • Missing alt text: Always include alt attributes for images
  • Using deprecated tags: Avoid old tags like <font> or <center>
  • Improper document structure: Include DOCTYPE, html, head, and body tags

Responsive Web Design

Responsive design ensures your website looks good on all devices. While this primarily involves CSS, HTML provides the foundation.

Viewport Meta Tag

Essential for responsive design:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Responsive Images

Use the <picture> element and srcset attribute for responsive images:

<img src="image-small.jpg" srcset="image-small.jpg 500w, image-medium.jpg 1000w, image-large.jpg 1500w" sizes="(max-width: 600px) 480px, (max-width: 1200px) 800px, 1200px" alt="Responsive image"> <picture> <source media="(max-width: 600px)" srcset="image-small.jpg"> <source media="(max-width: 1200px)" srcset="image-medium.jpg"> <img src="image-large.jpg" alt="Responsive image"> </picture>
While responsive design mainly uses CSS, proper HTML structure is crucial for it to work effectively.

Mobile-First Approach

When building responsive websites:

  1. Start with mobile layout (simplest version)
  2. Use semantic HTML for content structure
  3. Add media queries for larger screens
  4. Test on various devices

How to Practice HTML

Consistent practice is key to mastering HTML. Here are some effective ways to practice:

1. Build Small Projects

  • Personal portfolio page
  • Recipe page
  • Event invitation
  • Blog post

2. Code Along With Tutorials

Follow along with video tutorials and online courses, typing the code yourself rather than copying.

3. Recreate Existing Websites

Try to recreate simple websites you like to understand their structure.

4. Use Online Code Editors

Platforms like CodePen, JSFiddle, and CodeSandbox let you practice without setting up a local environment.

5. Contribute to Open Source

Look for beginner-friendly open source projects to contribute HTML improvements.

Don't just copy code‚Äîmake sure you understand what each element and attribute does.

HTML Practice Exercises

Try these exercises to test your HTML skills:

  1. Create a page with your favorite recipe (use headings, lists, and images)
  2. Build a simple contact form with name, email, and message fields
  3. Make a personal profile page with sections for about, skills, and contact
  4. Create a product card with image, title, description, and price
  5. Build a responsive navigation menu that works on mobile and desktop

Next Steps in Your Learning Journey

After mastering HTML basics, here's what to learn next:

1. CSS for Styling

Learn to make your HTML beautiful with colors, layouts, and responsive designs.

2. JavaScript for Interactivity

Add functionality to your websites with JavaScript.

3. Version Control with Git

Learn to track changes and collaborate with others using Git and GitHub.

4. Frontend Frameworks

Explore popular frameworks like React, Vue, or Angular for building complex applications.

5. Backend Development

Learn server-side programming with Node.js, Python, PHP, or other languages.

Remember that HTML is just the beginning‚Äîmodern web development involves many technologies working together.

Recommended Learning Path

  4. Responsive Design: Mobile-friendly websites
