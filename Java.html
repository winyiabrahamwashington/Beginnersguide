What is Java?

The powerful, platform-independent programming language

The Java Programming Language

Java is a high-level, class-based, object-oriented programming language that is designed to have as few implementation dependencies as possible. It is a general-purpose programming language intended to let programmers write once, run anywhere (WORA), meaning that compiled Java code can run on all platforms that support Java without the need for recompilation.

Key Features of Java

  • Platform Independent: Java code is compiled into bytecode that runs on any Java Virtual Machine (JVM)
  • Object-Oriented: Follows OOP paradigm for better code organization
  • Robust: Strong memory management, exception handling, and type checking
  • Secure: Provides a secure execution environment
  • Multithreaded: Supports concurrent execution of program parts
  • High Performance: Just-In-Time (JIT) compilers enable high performance
  • Distributed: Designed for distributed computing environments
  • Dynamic: Supports dynamic loading of classes and reflection

Understanding the Java Ecosystem

Java consists of several key components that work together:

Java Platform Components

  • Java Development Kit (JDK): Contains tools for developing Java programs (compiler, debugger, etc.)
  • Java Runtime Environment (JRE): Needed to run Java programs (includes JVM and core libraries)
  • Java Virtual Machine (JVM): Executes Java bytecode, enables "Write Once, Run Anywhere"
  • Java API: Extensive collection of ready-made classes for common programming tasks

How Java Works: Java source code (.java files) is compiled by the javac compiler into bytecode (.class files). The JVM then interprets this bytecode, converting it into native machine code at runtime using the JIT compiler.

// Simple Java program
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Did you know? Java was originally developed by James Gosling at Sun Microsystems (now owned by Oracle) and released in 1995. The name "Java" was inspired by Java coffee, marking the language's ability to "wake up" the web.

Getting Started with Java

Set up your development environment and write your first program

Installing Java Development Kit (JDK)

To start programming in Java, you'll need to install the Java Development Kit (JDK) which includes the Java Runtime Environment (JRE) and developer tools.

Installation Steps

  1. Download the latest JDK from Oracle's website or use OpenJDK
  2. Run the installer for your operating system
  3. Set the JAVA_HOME environment variable to your JDK installation path
  4. Add the JDK's bin directory to your system PATH
  5. Verify installation by running java -version in your terminal

JDK vs JRE

The JDK is for developers (contains compiler and tools) while the JRE is for running Java programs. When installing Java for development, always choose the JDK.

Beginner Tip

If you're having trouble with environment variables, many modern IDEs (like IntelliJ IDEA) can automatically detect your JDK installation without manual PATH configuration.

Your First Java Program

Let's create the traditional "Hello World" program to understand Java's basic structure.

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Program Breakdown

  • public class HelloWorld: Defines a class named HelloWorld (filename must match)
  • public static void main(String[] args): The main method - program entry point
  • System.out.println(): Prints output to the console
  • //: Single-line comment
  • /* */: Multi-line comment
  • /** */: Documentation comment (for Javadoc)

Important: The filename must match the class name (HelloWorld.java) and is case-sensitive. Java is a case-sensitive language.

Compiling and Running

To compile and run your program:

  1. Save the file as HelloWorld.java
  2. Open terminal in the file's directory
  3. Compile with: javac HelloWorld.java
  4. Run with: java HelloWorld

You should see "Hello, World!" printed in your terminal.

  • Mismatching class name and filename
  • Missing semicolons at line endings
  • Forgetting to declare variable types
  • Not properly closing blocks with }
  • Using the wrong type of quotes (should be double quotes for Strings)

Variables and Data Types

Java is a statically-typed language, meaning variables must be declared with a specific type.

// Primitive data types
byte age = 30;                  // 8-bit integer (-128 to 127)
short year = 2023;              // 16-bit integer (-32,768 to 32,767)
int population = 8000000;       // 32-bit integer (-2^31 to 2^31-1)
long globalPopulation = 7800000000L; // 64-bit integer (add L suffix)
float price = 19.99f;           // 32-bit floating point (add f suffix)
double pi = 3.1415926535;       // 64-bit floating point (default for decimals)
char grade = 'A';               // Single 16-bit Unicode character
boolean isJavaFun = true;       // true or false

// Non-primitive data types
String name = "Java Beginner";  // Sequence of characters (note capital S)
int[] numbers = {1, 2, 3};      // Array of integers
Object obj = new Object();      // Any Java object

// Constants (final variables)
final double PI = 3.14159;

Naming Conventions: Use camelCase for variables (myVariableName), PascalCase for classes (MyClassName), and UPPER_CASE for constants (MAX_VALUE). Choose descriptive names that indicate the variable's purpose.

Beginner Tip

Start with int for whole numbers and double for decimal numbers unless you have a specific reason to use other types. These are the most commonly used numeric types in Java.

Type Casting

Java supports both widening (automatic) and narrowing (manual) type conversions:

// Widening conversion (automatic)
int myInt = 9;
double myDouble = myInt;  // Automatic casting: int to double

// Narrowing conversion (manual)
double myDouble = 9.78;
int myInt = (int) myDouble; // Manual casting: double to int

Operators

Java provides various operators for performing operations on variables and values.

// Arithmetic operators
int sum = 10 + 20;      // Addition
int difference = 20 - 10; // Subtraction
int product = 10 * 20;  // Multiplication
int quotient = 20 / 10; // Division
int remainder = 11 % 3; // Modulus (remainder)

// Assignment operators
int x = 10;
x += 5;  // Equivalent to x = x + 5
x -= 3;  // Equivalent to x = x - 3
x *= 2;  // Equivalent to x = x * 2
x /= 4;  // Equivalent to x = x / 4

// Comparison operators
boolean isEqual = (10 == 20);    // Equal to
boolean isNotEqual = (10 != 20); // Not equal to
boolean isGreater = (10 > 20);   // Greater than
boolean isLess = (10 < 20);      // Less than
boolean isGreaterOrEqual = (10 >= 20);

// Logical operators
boolean andResult = (true && false); // Logical AND
boolean orResult = (true || false);  // Logical OR
boolean notResult = !true;           // Logical NOT

// Ternary operator
int time = 20;
String result = (time < 18) ? "Good day." : "Good evening.";

Operator Precedence

Java evaluates expressions according to operator precedence (order of operations):

  1. Parentheses ()
  2. Unary operators ++ -- !
  3. Multiplicative * / %
  4. Additive + -
  5. Relational < > <= >=
  6. Equality == !=
  7. Logical AND &&
  8. Logical OR ||
  9. Assignment = += -= etc.

Common Operator Mistakes

  • Using = (assignment) instead of == (equality)
  • Forgetting that integer division truncates decimals
  • Not understanding operator precedence
  • Using & (bitwise AND) instead of && (logical AND)
  • Confusing prefix (++x) and postfix (x++) increment

Control Flow Statements

Control flow statements allow you to control the execution flow of your program.

// If-else statement
int age = 18;
if (age < 13) {
    System.out.println("Child");
} else if (age < 20) {
    System.out.println("Teenager");
} else {
    System.out.println("Adult");
}

// Switch statement (Java 14+ enhanced)
String day = "Monday";
switch (day) {
    case "Monday" -> System.out.println("Start of work week");
    case "Friday" -> System.out.println("End of work week");
    default -> System.out.println("Midweek day");
}

// Traditional for loop
for (int i = 0; i < 5; i++) {
    System.out.println("Iteration: " + i);
}

// Enhanced for loop (for-each)
int[] numbers = {1, 2, 3, 4, 5};
for (int num : numbers) {
    System.out.println(num);
}

// While loop
int count = 0;
while (count < 5) {
    System.out.println("Count: " + count);
    count++;
}

// Do-while loop
int num = 1;
do {
    System.out.println("Number: " + num);
    num++;
} while (num <= 5);

// Break and continue
for (int i = 0; i < 10; i++) {
    if (i == 4) {
        continue; // Skip iteration
    }
    if (i == 8) {
        break; // Exit loop
    }
    System.out.println(i);
}

Enhanced For Loop: Java provides a simplified for loop for iterating through arrays and collections: for (String item : items) { ... }

Control Flow Tips

When working with control flow:

  • Always use braces {} even for single statements
  • Indent code properly for readability
  • Use break in switch statements to prevent fall-through
  • Consider using enhanced for loops when possible
  • Avoid deep nesting of control structures
  • Use meaningful conditions in loops and if statements

  2. Create a calculator program that takes two numbers and an operator (+, -, *, /) as input and outputs the result

Object-Oriented Programming in Java

Learn the core concepts of OOP with Java

Classes and Objects

Java is fundamentally object-oriented. A class is a blueprint for objects, and an object is an instance of a class.

// Class definition
public class Car {
    // Fields (attributes)
    private String brand;
    private String model;
    private int year;
    
    // Constructor
    public Car(String brand, String model, int year) {
        this.brand = brand;
        this.model = model;
        this.year = year;
    }
    
    // Getter methods
    public String getBrand() { return brand; }
    public String getModel() { return model; }
    public int getYear() { return year; }
    
    // Setter methods
    public void setBrand(String brand) { this.brand = brand; }
    public void setModel(String model) { this.model = model; }
    public void setYear(int year) { this.year = year; }
    
    // Method
    public void startEngine() {
        System.out.println("Starting the " + brand + " " + model);
    }
    
    // Method with return value
    public int calculateAge(int currentYear) {
        return currentYear - year;
    }
}

// Creating and using objects
public class Main {
    public static void main(String[] args) {
        // Create Car object
        Car myCar = new Car("Toyota", "Camry", 2020);
        
        // Call methods
        myCar.startEngine();  // Output: Starting the Toyota Camry
        
        // Access fields through getters
        System.out.println("Brand: " + myCar.getBrand());
        
        // Calculate car age
        int carAge = myCar.calculateAge(2023);
        System.out.println("Car age: " + carAge + " years");
    }
}

Class Components

  • Fields: Variables that hold the object's state (should usually be private)
  • Constructor: Special method for object initialization (same name as class)
  • Methods: Functions that define object behavior
  • Getters/Setters: Public methods to access private fields
  • this keyword: Refers to the current object
  • new keyword: Creates a new object instance

Best Practice: Follow the principle of encapsulation by making fields private and providing public getter/setter methods to access them. This protects the object's internal state.

Object-Oriented Thinking

When designing classes:

  1. Identify real-world objects your program needs
  2. Determine each object's attributes (fields) and behaviors (methods)
  3. Define relationships between objects
  4. Keep classes focused on a single responsibility
  5. Use meaningful names that reflect the object's purpose

Four Pillars of OOP

Object-Oriented Programming in Java is built on four main principles:

1. Encapsulation

Bundling data and methods that operate on that data within one unit (class) and restricting access to some of the object's components.

public class BankAccount {
    private double balance;  // Private field
    
    // Public methods to interact with balance
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
    
    public double getBalance() {
        return balance;
    }
}

Benefits: Prevents unauthorized access, allows validation, makes code more maintainable

2. Inheritance

Creating new classes (subclasses) from existing ones (superclasses) to reuse code and establish relationships.

// Superclass
class Vehicle {
    protected String brand;  // Protected allows access by subclasses
    
    public Vehicle(String brand) {
        this.brand = brand;
    }
    
    public void honk() {
        System.out.println("Honk honk!");
    }
}

// Subclass
class Car extends Vehicle {
    private String model;
    
    public Car(String brand, String model) {
        super(brand);  // Call superclass constructor
        this.model = model;
    }
    
    public void displayInfo() {
        System.out.println("Brand: " + brand + ", Model: " + model);
    }
}

// Using inheritance
public class Main {
    public static void main(String[] args) {
        Car myCar = new Car("Toyota", "Camry");
        myCar.honk();       // Inherited method
        myCar.displayInfo(); // Subclass method
    }
}

Benefits: Code reuse, logical hierarchy, polymorphism

3. Polymorphism

The ability of an object to take on many forms, typically through method overriding and interfaces.

class Animal {
    public void makeSound() {
        System.out.println("Some sound");
    }
}

class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Bark");
    }
}

class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Meow");
    }
}

// Using polymorphism
public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Animal();  // Animal object
        Animal myDog = new Dog();        // Dog object
        Animal myCat = new Cat();        // Cat object
        
        myAnimal.makeSound();  // Output: Some sound
        myDog.makeSound();     // Output: Bark
        myCat.makeSound();     // Output: Meow
    }
}

Benefits: Flexible code, easier to extend, cleaner interfaces

4. Abstraction

Hiding complex implementation details and showing only the necessary features of an object.

// Abstract class
abstract class Shape {
    // Abstract method (no implementation)
    abstract double area();
    
    // Concrete method
    void display() {
        System.out.println("Area: " + area());
    }
}

// Concrete subclass
class Circle extends Shape {
    private double radius;
    
    Circle(double r) { radius = r; }
    
    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

// Using abstraction
public class Main {
    public static void main(String[] args) {
        Shape myCircle = new Circle(5.0);
        myCircle.display();  // Output: Area: 78.53981633974483
    }
}

Benefits: Simplifies complex systems, reduces complexity, focuses on what rather than how

SOLID Principles: Beyond the four pillars, good OOP design follows SOLID principles:

  1. Single Responsibility: A class should have one reason to change
  2. Open-Closed: Open for extension, closed for modification
  3. Liskov Substitution: Subclasses should be substitutable for their base classes
  4. Interface Segregation: Many specific interfaces are better than one general one
  5. Dependency Inversion: Depend on abstractions, not concretions

Interfaces and Abstract Classes

Interfaces and abstract classes are key tools for implementing abstraction in Java.

// Interface definition
interface Drawable {
    void draw();  // Abstract method
    
    default void resize() {  // Default method (Java 8+)
        System.out.println("Resizing");
    }
    
    static void printInfo() {  // Static method (Java 8+)
        System.out.println("This is a Drawable interface");
    }
}

// Abstract class
abstract class Shape {
    protected String color;
    
    public Shape(String color) {
        this.color = color;
    }
    
    // Abstract method
    abstract double area();
    
    // Concrete method
    public String getColor() {
        return color;
    }
}

// Concrete class implementing interface and extending abstract class
class Circle extends Shape implements Drawable {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public void draw() {
        System.out.println("Drawing a " + color + " circle");
    }
    
    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

// Using interface and abstract class
public class Main {
    public static void main(String[] args) {
        Circle myCircle = new Circle("red", 5.0);
        myCircle.draw();          // Output: Drawing a red circle
        myCircle.resize();        // Output: Resizing
        System.out.println("Area: " + myCircle.area());
        Drawable.printInfo();     // Output: This is a Drawable interface
    }
}

Interfaces vs Abstract Classes

Feature Interface Abstract Class
Methods Abstract (default/static in Java 8+) Can have both abstract and concrete
Fields Only constants (public static final) Can have any fields
Constructors No Yes
Multiple Inheritance Class can implement multiple Class can extend only one
Access Modifiers Public by default Can have any access modifier

Interface Tips

When working with interfaces:

  • Use interfaces to define behavior contracts
  • A class can implement multiple interfaces
  • Java 8+ allows default and static methods in interfaces
  • Prefer interfaces to abstract classes when possible
  • Use the @Override annotation when implementing interface methods

Functional Interfaces: Java 8 introduced functional interfaces (interfaces with a single abstract method) that enable lambda expressions, making Java more functional.

// Functional interface
@FunctionalInterface
interface Greeting {
    void sayHello(String name);
}

// Using lambda expression
public class Main {
    public static void main(String[] args) {
        Greeting greet = name -> System.out.println("Hello, " + name);
        greet.sayHello("Alice");  // Output: Hello, Alice
    }
}

Collections Framework

The Java Collections Framework provides a set of interfaces and classes for storing and manipulating groups of data.

import java.util.*;

public class CollectionsDemo {
    public static void main(String[] args) {
        // List - ordered collection (allows duplicates)
        List arrayList = new ArrayList<>();
        arrayList.add("Apple");
        arrayList.add("Banana");
        arrayList.add(1, "Orange"); // Insert at index 1
        
        // Set - unordered collection (no duplicates)
        Set hashSet = new HashSet<>();
        hashSet.add(10);
        hashSet.add(20);
        hashSet.add(10); // Duplicate, won't be added
        
        // Map - key-value pairs
        Map hashMap = new HashMap<>();
        hashMap.put("John", 25);
        hashMap.put("Alice", 30);
        hashMap.put("Bob", 28);
        
        // Queue - FIFO order
        Queue queue = new LinkedList<>();
        queue.add("First");
        queue.add("Second");
        queue.add("Third");
        
        // Iterating through collections
        System.out.println("ArrayList:");
        for (String fruit : arrayList) {
            System.out.println(fruit);
        }
        
        System.out.println("\nHashMap:");
        for (Map.Entry entry : hashMap.entrySet()) {
            System.out.println(entry.getKey() + ": " + entry.getValue());
        }
    }
}

Common Collection Interfaces

  • List: Ordered collection (ArrayList, LinkedList)
  • Set: Collection with no duplicates (HashSet, TreeSet)
  • Map: Key-value pairs (HashMap, TreeMap)
  • Queue: FIFO ordering (LinkedList, PriorityQueue)
  • Deque: Double-ended queue (ArrayDeque)

Generics: Java collections use generics to provide type safety. Always specify the type when declaring collections: List<String> rather than raw List.

Exception Handling

Java provides a robust exception handling mechanism to deal with runtime errors.

import java.io.File;
import java.io.FileNotFoundException;
import java.util.Scanner;

public class ExceptionHandling {
    public static void main(String[] args) {
        // Try-catch block
        try {
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]); // ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Array index out of bounds!");
        }
        
        // Multiple catch blocks
        try {
            File file = new File("nonexistent.txt");
            Scanner scanner = new Scanner(file); // FileNotFoundException
        } catch (FileNotFoundException e) {
            System.out.println("File not found: " + e.getMessage());
        } catch (Exception e) {
            System.out.println("Something went wrong");
        }
        
        // Finally block
        Scanner fileScanner = null;
        try {
            File file = new File("example.txt");
            fileScanner = new Scanner(file);
            while (fileScanner.hasNextLine()) {
                System.out.println(fileScanner.nextLine());
            }
        } catch (FileNotFoundException e) {
            System.out.println("File not found");
        } finally {
            if (fileScanner != null) {
                fileScanner.close(); // Always executed
            }
        }
        
        // Try-with-resources (Java 7+)
        try (Scanner autoCloseScanner = new Scanner(new File("example.txt"))) {
            while (autoCloseScanner.hasNextLine()) {
                System.out.println(autoCloseScanner.nextLine());
            }
        } catch (FileNotFoundException e) {
            System.out.println("File not found");
        } // Scanner automatically closed
        
        // Throwing exceptions
        try {
            validateAge(15);
        } catch (InvalidAgeException e) {
            System.out.println(e.getMessage());
        }
    }
    
    // Custom exception
    static class InvalidAgeException extends Exception {
        public InvalidAgeException(String message) {
            super(message);
        }
    }
    
    // Method throwing exception
    static void validateAge(int age) throws InvalidAgeException {
        if (age < 18) {
            throw new InvalidAgeException("Age must be 18 or older");
        }
    }
}

Exception Hierarchy

  • Throwable: Superclass of all errors and exceptions
  • Error: Serious problems that shouldn't be caught (OutOfMemoryError)
  • Exception: Problems that can be handled
  • RuntimeException: Unchecked exceptions (NullPointerException)
  • Checked Exceptions: Must be declared or caught (IOException)

Best Practices:

  • Use specific exception types rather than generic Exception
  • Don't catch exceptions you can't handle
  • Clean up resources in finally blocks or use try-with-resources
  • Document exceptions your methods throw with @throws
  • Create meaningful custom exceptions when appropriate

File I/O and Streams

Java provides comprehensive APIs for working with files and streams.

import java.io.*;
import java.nio.file.*;
import java.util.stream.*;

public class FileIO {
    public static void main(String[] args) {
        // Writing to a file (traditional way)
        try (FileWriter writer = new FileWriter("output.txt")) {
            writer.write("Hello, File I/O!\n");
            writer.write("This is a second line.\n");
        } catch (IOException e) {
            System.out.println("Error writing to file: " + e.getMessage());
        }
        
        // Reading from a file (traditional way)
        try (BufferedReader reader = new BufferedReader(new FileReader("output.txt"))) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            System.out.println("Error reading file: " + e.getMessage());
        }
        
        // New Java NIO API (Java 7+)
        try {
            // Write file
            Path path = Paths.get("nio_output.txt");
            Files.write(path, "Hello NIO!\n".getBytes(), StandardOpenOption.CREATE);
            Files.write(path, "Second line\n".getBytes(), StandardOpenOption.APPEND);
            
            // Read file
            List lines = Files.readAllLines(path);
            for (String line : lines) {
                System.out.println(line);
            }
            
            // Stream API with files
            try (Stream stream = Files.lines(path)) {
                stream.forEach(System.out::println);
            }
        } catch (IOException e) {
            System.out.println("NIO error: " + e.getMessage());
        }
        
        // Object serialization
        try (ObjectOutputStream oos = new ObjectOutputStream(
                new FileOutputStream("person.ser"))) {
            Person person = new Person("Alice", 30);
            oos.writeObject(person);
        } catch (IOException e) {
            e.printStackTrace();
        }
        
        // Object deserialization
        try (ObjectInputStream ois = new ObjectInputStream(
                new FileInputStream("person.ser"))) {
            Person person = (Person) ois.readObject();
            System.out.println("Deserialized: " + person);
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}

// Serializable class
class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
}

Key I/O Concepts

  • Byte Streams: InputStream, OutputStream (for binary data)
  • Character Streams: Reader, Writer (for text data)
  • Buffered I/O: BufferedInputStream, BufferedReader (more efficient)
  • NIO (New I/O): Path, Files, Channels (non-blocking I/O)
  • Serialization: Converting objects to byte streams

Try-with-resources: Always use try-with-resources for I/O operations to ensure proper resource cleanup. This works with any class implementing AutoCloseable.

Multithreading

Java supports multithreading to perform multiple tasks simultaneously.

public class ThreadDemo {
    public static void main(String[] args) {
        // Extending Thread class
        Thread thread1 = new MyThread();
        thread1.start();
        
        // Implementing Runnable interface
        Thread thread2 = new Thread(new MyRunnable());
        thread2.start();
        
        // Lambda expression for Runnable
        Thread thread3 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("Lambda thread: " + i);
                try {
                    Thread.sleep(500);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });
        thread3.start();
        
        // Thread synchronization
        Counter counter = new Counter();
        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        });
        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 1000; i++) {
                counter.increment();
            }
        });
        t1.start();
        t2.start();
        try {
            t1.join();
            t2.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println("Final count: " + counter.getCount());
    }
}

class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Thread class: " + i);
            try {
                Thread.sleep(500);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println("Runnable: " + i);
            try {
                Thread.sleep(500);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

class Counter {
    private int count = 0;
    
    public synchronized void increment() {
        count++;
    }
    
    public int getCount() {
        return count;
    }
}

Thread Concepts

  • Thread: Lightweight process that executes code
  • Runnable: Interface representing a task to execute
  • Synchronization: Controlling access to shared resources
  • Thread States: New, Runnable, Blocked, Waiting, Timed Waiting, Terminated
  • Thread Pool: Managing multiple threads efficiently

Concurrency Issues: Be aware of race conditions, deadlocks, and thread interference when working with multiple threads. Always synchronize access to shared resources.

Java Concurrency API: For more advanced threading, explore:

  • ExecutorService and ThreadPools
  • Callable and Future
  • Concurrent collections (ConcurrentHashMap)
  • Atomic variables
  • Locks and Conditions
