What is PHP?

Server-side scripting language for web development

The Backbone of the Web

PHP (Hypertext Preprocessor) is a widely-used open source server-side scripting language designed for web development. Created by Rasmus Lerdorf in 1994, PHP has evolved into a powerful language that powers major platforms like WordPress, Facebook (originally), and Wikipedia.

Key Features of PHP

  • Server-side: Executes on the server before sending to browser
  • Cross-platform: Runs on Windows, Linux, macOS, etc.
  • Database integration: Works with MySQL, PostgreSQL, MongoDB
  • Easy to learn: Simple syntax similar to C and Java
  • Large community: Extensive documentation and support
  • Frameworks: Laravel, Symfony, CodeIgniter, etc.
<?php
// Simple PHP example
echo "Hello, World!";

// Variables
$name = "Alice";
$age = 30;

// String interpolation
echo "My name is $name and I'm $age years old.";

// Arrays
$colors = array("red", "green", "blue");
echo $colors[0]; // Outputs "red"

// Functions

    return "Hello, $name!";
}

Did you know? PHP is used by 78% of all websites whose server-side programming language is known. It's particularly dominant in content management systems like WordPress, which powers over 40% of all websites.

PHP Syntax Basics

Fundamental concepts every PHP developer needs to know

Variables and Constants

PHP is loosely typed and uses dollar signs ($) for variables. Constants are defined differently.

<?php
// Variables (can change)
$name = "Alice";
$age = 30;
$price = 9.99;
$isStudent = true;

// Variable variables
$varName = "age";
echo $$varName; // Outputs 30

// Constants (can't change)
define("PI", 3.14159);
const SITE_NAME = "My Website";

// Magic constants
echo __LINE__; // Current line number
echo __FILE__; // Full path and filename
echo __DIR__; // Directory of the file

// Variable scope
function test() {
    global $name; // Access global variable
    $localVar = "I'm local"; // Local variable
}

// Variable checking
isset($name); // Check if variable exists
empty($name); // Check if variable is empty
?>

Naming Conventions: PHP variables use camelCase or snake_case. Constants are traditionally UPPERCASE. Use descriptive names that explain their purpose. Variable names are case-sensitive.

Variables and Constants Explained

Key Differences
Variables Constants

Best Practices
  • Initialize variables when you declare them
  • Use constants for values that shouldn't change
  • Avoid global variables when possible
  • Use meaningful, descriptive names
Common Mistakes

Data Types

PHP has several built-in data types that form the building blocks of your programs.

<?php
// Scalar types

// Compound types

is_int($integer); // true

Control Structures

Control the flow of your program's execution with conditionals and loops.

<?php
// If-elseif-else
$age = 25;

if ($age < 18) {
    echo "Child";
} elseif ($age < 65) {
    echo "Adult";
} else {
    echo "Senior";
}

// Switch statement
$day = "Monday";

switch ($day) {
    case "Monday":
        echo "Start of work week";
        break;
    case "Friday":
        echo "Almost weekend!";
        break;
    default:
        echo "Regular day";
}

// While loop
$count = 1;
while ($count <= 5) {
    echo $count++;
}

// Do-while loop
$count = 1;
do {
    echo $count++;
} while ($count <= 5);

// For loop
for ($i = 0; $i < 5; $i++) {
    echo $i;
}

// Foreach loop (for arrays)
$colors = ["red", "green", "blue"];
foreach ($colors as $color) {
    echo $color;
}

// Foreach with keys
$person = ["name" => "Alice", "age" => 30];
foreach ($person as $key => $value) {
    echo "$key: $value";
}

// Alternative syntax for control structures
if ($age > 18):
    echo "Adult";
else:
    echo "Child";
endif;
?>

Alternative Syntax: PHP offers alternative syntax for control structures using colons and endif, endwhile, endfor, etc. This is particularly useful when mixing PHP with HTML in templates.

Control Structures Explained

Control structures are how your program makes decisions and repeats actions:

Conditional Statements
  • if: Executes code if condition is true
  • if-else: Provides alternative path when condition is false
  • switch: Checks multiple possible values
  • ternary operator: Short form for simple if-else
Looping Constructs
  • while: Repeats while condition is true
  • do-while: Executes at least once, then checks condition
  • for: Repeats specific number of times
  • foreach: Iterates through arrays
Control Transfer
  • break: Exit loop or switch
  • continue: Skip to next iteration
  • return: Exit function with optional value

Functions

Functions allow you to organize code into reusable blocks.

<?php
// Basic function
function greet($name) {
    return "Hello, $name!";
}
echo greet("Alice");

// Function with default parameter
function sayHello($name = "Guest") {
    echo "Hello, $name!";
}
sayHello(); // Uses default
sayHello("Bob");

// Type declarations (PHP 7+)
function addNumbers(float $a, float $b): float {
    return $a + $b;
}
echo addNumbers(1.5, 2.5);

// Variable functions
$func = "greet";
echo $func("Alice");

// Anonymous functions (closures)
$greet = function($name) {
    echo "Hello, $name!";
};
$greet("Charlie");

// Arrow functions (PHP 7.4+)
$multiply = fn($a, $b) => $a * $b;
echo $multiply(5, 3);

// Variable number of arguments
function sum(...$numbers) {
    return array_sum($numbers);
}
echo sum(1, 2, 3, 4);

// Strict typing (PHP 7+)
declare(strict_types=1);
function strictAdd(int $a, int $b): int {
    return $a + $b;
}
?>

Strict Typing: PHP 7+ supports type declarations and strict typing. Use declare(strict_types=1); at the top of your file to enable strict type checking, which can help catch bugs early.

Working with Forms

PHP is commonly used to process HTML form data submitted by users.

<?php
// form.php - form processing example
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Get form data
    $name = $_POST['name'] ?? '';
    $email = $_POST['email'] ?? '';
    $message = $_POST['message'] ?? '';
    
    // Validate data
    $errors = [];
    
    if (empty($name)) {
        $errors[] = 'Name is required';
    }
    
    if (empty($email) || !filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errors[] = 'Valid email is required';
    }
    
    if (empty($message)) {
        $errors[] = 'Message is required';
    }
    
    // Process if no errors
    if (empty($errors)) {
        // Save to database, send email, etc.
        $success = true;
    }
}
?>

<!DOCTYPE html>
<html>
<head>
    <title>Contact Form</title>
</head>
<body>
    <?php if (!empty($errors)): ?>
        <div class="errors">
            <?php foreach ($errors as $error): ?>
                <p><?php echo htmlspecialchars($error); ?></p>
            <?php endforeach; ?>
        </div>
    <?php endif; ?>
    
    <?php if (isset($success) && $success): ?>
        <div class="success">Thank you for your message!</div>
    <?php else: ?>
        <form method="post">
            <label for="name">Name:</label>
            <input type="text" id="name" name="name" value="<?php echo htmlspecialchars($name ?? ''); ?>">
            
            <label for="email">Email:</label>
            <input type="email" id="email" name="email" value="<?php echo htmlspecialchars($email ?? ''); ?>">
            
            <label for="message">Message:</label>
            <textarea id="message" name="message"><?php echo htmlspecialchars($message ?? ''); ?></textarea>
            
            <button type="submit">Send</button>
        </form>
    <?php endif; ?>
</body>
</html>

Security Note: Always sanitize and validate user input. Use htmlspecialchars() when outputting user-provided data to prevent XSS attacks. For database operations, use prepared statements to prevent SQL injection.

Working with Databases

PHP can connect to various databases, with MySQL being the most common choice.

<?php
// Database connection with PDO
try {
    $pdo = new PDO(
        'mysql:host=localhost;dbname=my_database;charset=utf8',
        'username',
        'password',
        [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
        ]
    );
} catch (PDOException $e) {
    die("Connection failed: " . $e->getMessage());
}

// Prepared statement (safe from SQL injection)
$stmt = $pdo->prepare("SELECT * FROM users WHERE email = :email");
$stmt->execute(['email' => $email]);
$user = $stmt->fetch();

// Insert data
$stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (?, ?)");
$stmt->execute([$name, $email]);
$newId = $pdo->lastInsertId();

// Update data
$stmt = $pdo->prepare("UPDATE users SET name = ? WHERE id = ?");
$stmt->execute([$newName, $userId]);

// Delete data
$stmt = $pdo->prepare("DELETE FROM users WHERE id = ?");
$stmt->execute([$userId]);

// Fetch multiple rows
$stmt = $pdo->query("SELECT * FROM users");
$users = $stmt->fetchAll();

// Transactions
try {
    $pdo->beginTransaction();
    
    // Multiple operations
    $pdo->exec("UPDATE accounts SET balance = balance - 100 WHERE user_id = 1");
    $pdo->exec("UPDATE accounts SET balance = balance + 100 WHERE user_id = 2");
    
    $pdo->commit();
} catch (Exception $e) {
    $pdo->rollBack();
    echo "Transaction failed: " . $e->getMessage();
}
?>

PDO vs MySQLi: PHP offers two main extensions for MySQL: PDO (supports multiple databases) and MySQLi (MySQL only). PDO is generally preferred for its flexibility and named parameters. Always use prepared statements to prevent SQL injection.

Sample CRUD Operations
<?php
// Create (Insert)
function createUser(PDO $pdo, string $name, string $email): int {
    $stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (?, ?)");
    $stmt->execute([$name, $email]);
    return $pdo->lastInsertId();
}

// Read (Select)
function getUser(PDO $pdo, int $id): ?array {
    $stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
    $stmt->execute([$id]);
    return $stmt->fetch() ?: null;
}

// Update
function updateUser(PDO $pdo, int $id, string $name, string $email): bool {
    $stmt = $pdo->prepare("UPDATE users SET name = ?, email = ? WHERE id = ?");
    return $stmt->execute([$name, $email, $id]);
}

// Delete
function deleteUser(PDO $pdo, int $id): bool {
    $stmt = $pdo->prepare("DELETE FROM users WHERE id = ?");
    return $stmt->execute([$id]);
}

// Usage example
$userId = createUser($pdo, "Alice", "alice@example.com");
$user = getUser($pdo, $userId);
updateUser($pdo, $userId, "Alice Smith", "alice.smith@example.com");
deleteUser($pdo, $userId);
?>

Sessions and Cookies

Sessions and cookies allow you to persist data across multiple page requests.

<?php
// Starting a session
session_start();

// Setting session data
$_SESSION['user_id'] = 123;
$_SESSION['username'] = 'alice';

// Accessing session data
if (isset($_SESSION['user_id'])) {
    echo "Welcome back, " . $_SESSION['username'];
}

// Removing session data
unset($_SESSION['username']);

// Destroying a session
session_destroy();

// Setting a cookie
setcookie(
    'preference',       // Name
    'dark_mode',        // Value
    time() + 86400 * 30, // Expiry (30 days)
    '/',                // Path
    '',                 // Domain
    true,               // Secure (HTTPS only)
    true                // HttpOnly (no JS access)
);

// Accessing cookies
if (isset($_COOKIE['preference'])) {
    echo "Your preference: " . $_COOKIE['preference'];
}

// Deleting a cookie
setcookie('preference', '', time() - 3600, '/');
?>

Security Note: Never store sensitive data in cookies. Use sessions for sensitive data, as they're stored on the server. For sessions, regenerate the ID after login to prevent session fixation attacks (session_regenerate_id(true)).

Sessions vs Cookies

Understanding the difference between sessions and cookies is important:

Sessions
  • Stored on the server
  • More secure for sensitive data
  • Temporary (until browser closes or expires)
  • Uses a session ID cookie
Cookies
  • Stored on the client's browser
  • Less secure (can be modified)
  • Can persist after browser closes
  • Limited size (about 4KB)
When to Use Each
Use Case Technology
User login state Session
User preferences Cookie
Shopping cart Session
Tracking/analytics Cookie
Security Best Practices
  • Use session_start() before any output
  • Regenerate session ID after login
  • Set cookies with HttpOnly and Secure flags
  • Validate and sanitize all session/cookie data

File Handling

PHP provides functions for working with files and directories on the server.

<?php
// Writing to a file
file_put_contents('data.txt', "Hello, PHP!");

// Reading a file
$content = file_get_contents('data.txt');
echo $content;

// File handling with fopen
$file = fopen('data.txt', 'a+'); // Open for reading and writing
fwrite($file, "\nNew line of text");
rewind($file); // Rewind to start
echo fread($file, filesize('data.txt'));
fclose($file);

// Reading line by line
$file = fopen('largefile.txt', 'r');
while (!feof($file)) {
    echo fgets($file) . "<br>";
}
fclose($file);

// File information
echo file_exists('data.txt') ? "Exists" : "Doesn't exist";
echo is_file('data.txt') ? "Is a file" : "Not a file";
echo is_dir('folder') ? "Is a directory" : "Not a directory";
echo filesize('data.txt') . " bytes";

// Directory operations
mkdir('new_folder');
rmdir('empty_folder');

// Scan directory
$files = scandir('.');
foreach ($files as $file) {
    if ($file !== '.' && $file !== '..') {
        echo "$file<br>";
    }
}

// File upload handling
if ($_SERVER['REQUEST_METHOD'] === 'POST' && isset($_FILES['file'])) {
    $uploadDir = 'uploads/';
    $uploadFile = $uploadDir . basename($_FILES['file']['name']);
    
    if (move_uploaded_file($_FILES['file']['tmp_name'], $uploadFile)) {
        echo "File uploaded successfully";
    } else {
        echo "File upload failed";
    }
}
?>

Security Warning: Be extremely careful with file operations. Never use user-provided data directly in file paths without validation. For file uploads, restrict allowed types and scan for malware.

File Upload Best Practices
  1. Check $_FILES['file']['error'] for errors
  2. Verify file type with $_FILES['file']['type'] or file extension
  3. Use move_uploaded_file() to save the file
  4. Store outside web root or restrict access
  5. Set maximum file size limits
Common Use Cases
  • Configuration files
  • Data storage (when DB is overkill)
  • Log files
  • User uploads (images, documents)
