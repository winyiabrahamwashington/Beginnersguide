What is Ruby?

An elegant, object-oriented programming language

The Language of Developer Happiness

Ruby is a dynamic, open source programming language created by Yukihiro "Matz" Matsumoto in the mid-1990s in Japan. Designed for programmer productivity and fun, Ruby combines simple syntax with powerful object-oriented capabilities.

Key Features of Ruby

  • Object-Oriented: Everything in Ruby is an object
  • Dynamic Typing: No need to declare variable types
  • Readable Syntax: Designed to be natural to read and write
  • Flexible: Allows modification of its own parts
  • Garbage Collected: Automatic memory management
  • Rich Standard Library: Comes with many built-in methods
# Simple Ruby example
puts "Hello, Ruby!"

# Variables and constants
name = "Alice"  # Variable
AGE = 30        # Constant

# String interpolation
puts "My name is #{name} and I'm #{AGE} years old."

# Everything is an object
5.times { puts "Ruby is fun!" }

# Blocks and iterators
[1, 2, 3].each do |num|
  puts num * 2
end

Did you know? Ruby is most famous as the language behind the Ruby on Rails web framework, but it's also used for scripting, automation, data processing, and more. The language's motto is "Matz is nice and so we are nice" (MINASWAN).

Absolute Beginner's Guide to Ruby

If you're completely new to programming or Ruby, here's what you need to know to get started:

1. Understanding Programming Basics

Before diving into Ruby, it's helpful to understand some fundamental programming concepts:

  • Variables: Containers for storing data values
  • Constants: Values that don't change after being set
  • Data Types: Different kinds of data (numbers, text, true/false)
  • Control Flow: How your program makes decisions (if/else statements)
  • Loops: Repeating actions multiple times
  • Functions: Reusable blocks of code (called methods in Ruby)
  • Classes: Blueprints for creating objects
2. Ruby's Unique Features

Ruby has several features that make it beginner-friendly:

  • Everything is an object: Even primitive types like numbers
  • Flexible syntax: Multiple ways to accomplish the same thing
  • Blocks: Powerful way to pass around chunks of code
  • Mixins: Share functionality between classes
  • Metaprogramming: Code that writes code
3. Your First Steps in Ruby

To write your first Ruby program:

  1. Install Ruby on your computer
  2. Create a new file with a .rb extension
  3. Type the "Hello, World!" example above
  4. Run the program with ruby filename.rb
  5. Experiment by changing the text and running it again

Getting Started

Setting up your Ruby development environment

1. Install Ruby

Ruby comes pre-installed on macOS and many Linux distributions. For Windows or to get the latest version, you'll need to install it.

Steps to Install Ruby

  1. Windows: Download RubyInstaller from rubyinstaller.org
  2. macOS: Use Homebrew: brew install ruby
  3. Linux: Use your package manager (e.g., sudo apt-get install ruby-full on Ubuntu)
  4. Verify installation by running ruby -v in your terminal

Version Management: For serious Ruby development, consider using a version manager like RVM or rbenv. These tools let you switch between different Ruby versions easily.

Ruby Syntax Basics

Fundamental concepts every Ruby developer needs to know

Variables and Data Types

Ruby is dynamically typed, so you don't need to declare variable types.

# Variables (snake_case convention)
name = "Alice"  # String
age = 30        # Integer
price = 19.99   # Float
is_active = true # Boolean

# Constants (start with capital letter)
Pi = 3.14159

# Arrays
colors = ["red", "green", "blue"]
numbers = [1, 2, 3, 4, 5]

# Hashes (key-value pairs)
person = { name: "Alice", age: 30 } # Symbol keys (Ruby 1.9+ syntax)
person = { :name => "Alice", :age => 30 } # Older syntax

# Everything is an object
5.class # => Integer
"hello".class # => String
true.class # => TrueClass
nil.class # => NilClass

Naming Conventions: Ruby uses snake_case for variables and methods, CamelCase for classes and modules, and SCREAMING_SNAKE_CASE for constants. Use descriptive names that explain their purpose.

Variables and Data Types Explained

Understanding variables and data types is crucial for beginners:

Key Differences
Data Type Example Description
String "hello" Text data
Integer 42 Whole numbers
Float 3.14 Decimal numbers
Boolean true, false True/false values
Array [1, 2, 3] Ordered collection
Hash {a: 1, b: 2} Key-value pairs
Best Practices
  • Use descriptive variable names
  • Use constants for values that shouldn't change
  • Prefer symbols over strings for hash keys when possible
  • Use the new hash syntax ({key: value}) when possible
Common Mistakes
  • Using the wrong case for constants (must start with capital)
  • Using vague names like a or temp
  • Confusing symbols and strings

Control Flow

Control the flow of your program's execution with conditionals and loops.

# If-else statement
if age >= 18
  puts "You're an adult"
elsif age >= 13
  puts "You're a teenager"
else
  puts "You're a child"
end

# Unless statement (opposite of if)
unless age < 18
  puts "You can vote"
end

# Case statement
grade = 'B'
case grade
when 'A'
  puts "Excellent!"
when 'B'
  puts "Good job"
when 'C'
  puts "Needs improvement"
else
  puts "See me after class"
end

# While loop
count = 0
while count < 5
  puts count
  count += 1
end

# Until loop (opposite of while)
until count == 0
  puts count
  count -= 1
end

# For loop (less common in Ruby)
for i in 1..5
  puts i
end

# Each iterator (preferred over for loops)
(1..5).each do |i|
  puts i
end

# Ranges
1..10    # inclusive range
1...10   # exclusive range (1-9)
'a'..'d' # letter range

Ruby Style: In Ruby, we prefer using iterators like each rather than traditional for loops. Ruby provides many powerful iterators for different use cases.

Control Flow Explained

Control flow is how your program makes decisions and repeats actions:

Conditional Statements
  • if/elsif/else: Basic conditional execution
  • unless: Execute unless condition is true
  • case/when: Multiple condition checks
  • ternary operator: Short if/else (condition ? true_expr : false_expr)
Looping Constructs
  • while: Repeat while condition is true
  • until: Repeat until condition is true
  • for: Iterate over a range or collection
  • each: Preferred iterator for collections
Control Transfer
  • break: Exit loop immediately
  • next: Skip to next iteration
  • redo: Restart current iteration
  • return: Exit from method
Practical Examples
# Checking multiple conditions
if age >= 18 && has_id
  puts "You can vote"
else
  puts "You cannot vote"
end

# Looping through an array
names = ["Alice", "Bob", "Charlie"]
names.each do |name|
  puts "Hello, #{name}!"
end

# While loop with condition
counter = 0
while counter < 10
  puts counter
  counter += 1
end

# Case statement with ranges
score = 85
case score
when 90..100 then puts "A"
when 80..89  then puts "B"
when 70..79  then puts "C"
else              puts "F"
end

Methods

Methods are defined with the def keyword and can return values.

# Simple method
def greet(name)
  puts "Hello, #{name}!"
end

# Method with default argument
def say(message = "Hello")
  puts message
end

# Method with return value
def add(a, b)
  a + b  # Last expression is returned
end

# Method with explicit return
def max(a, b)
  return a if a > b
  b
end

# Method with variable arguments
def sum(*numbers)
  numbers.sum
end

# Calling methods
greet("Alice")
say
result = add(5, 7)
puts "Sum is #{result}"
puts "Max is #{max(10, 20)}"
puts "Sum is #{sum(1, 2, 3, 4, 5)}"

# Predicate methods (convention: end with ?)
def even?(number)
  number % 2 == 0
end

# Bang methods (convention: end with !)
def capitalize!(string)
  string.upcase!
end

Method Conventions: Ruby has naming conventions for methods: ? for predicate methods that return true/false, ! for methods that modify their receiver or raise errors, and = for setter methods.

Methods Explained

Methods are reusable blocks of code that perform specific tasks:

Method Components
  • def: Keyword to define a method
  • Name: Should be descriptive and lowercase
  • Parameters: Inputs the method accepts
  • Body: Code that executes when method is called
  • Return value: Last expression evaluated (or explicit return)
Special Method Types
  • Predicate methods: End with ? and return boolean
  • Bang methods: End with ! and modify receiver or raise errors
  • Setter methods: End with = for attribute assignment
Best Practices
  • Keep methods small and focused
  • Use descriptive names
  • Follow naming conventions (? ! =)
  • Limit side effects when possible
  • Document complex methods
Common Mistakes
  • Forgetting to call the method (just defining it)
  • Confusing return values (last expression is returned)
  • Using unclear method names

Blocks and Iterators

Blocks are chunks of code that can be passed to methods, a powerful Ruby feature.

# Block (used with methods)
[1, 2, 3].each { |n| puts n * 2 }

# Multi-line block
[1, 2, 3].each do |n|
  puts n * 2
end

# Yield keyword (method that accepts a block)
def three_times
  yield
  yield
  yield
end

three_times { puts "Hello" }

# Block parameters
def math_operation(a, b)
  yield(a, b)
end

result = math_operation(5, 3) { |x, y| x * y }
puts result # => 15

# Common iterators
numbers = [1, 2, 3, 4, 5]

# Map/collect
squares = numbers.map { |n| n * n }

# Select/find_all
evens = numbers.select { |n| n.even? }

# Reject
odds = numbers.reject { |n| n.even? }

# Reduce/inject
sum = numbers.reduce(0) { |total, n| total + n }

# Each with index
["a", "b", "c"].each_with_index do |item, index|
  puts "#{index}: #{item}"
end

Block Variables: Be careful with variable names in blocks. Variables defined in blocks can sometimes affect variables in the outer scope, especially in older Ruby versions. Use unique names to avoid confusion.

Blocks and Iterators Explained

Blocks are one of Ruby's most powerful features. Here's what beginners should know:

What Are Blocks?

Blocks are anonymous chunks of code that can be passed to methods. They can be enclosed in either curly braces {} or do...end.

Common Uses
  • Iteration: each, map, select, etc.
  • Resource management: Ensure resources are cleaned up
  • Custom behavior: Pass behavior into methods
Key Concepts
Concept Description Example
Block Chunk of code passed to method { |x| puts x }
Yield Execute the passed block yield(arg)
Proc Saved block (procedure) Proc.new { |x| x * 2 }
Lambda Stricter Proc with argument checking ->(x) { x * 2 }
Practical Examples
# File operations with blocks (auto-closes file)
File.open("example.txt", "w") do |file|
  file.puts "Hello, world!"
end

# Custom method with block
def benchmark
  start_time = Time.now
  yield
  end_time = Time.now
  puts "Time taken: #{end_time - start_time} seconds"
end

benchmark do
  # Some expensive operation
  (1..1000000).each { |n| n * n }
end

# Using procs
double = Proc.new { |n| n * 2 }
puts [1, 2, 3].map(&double)

# Using lambdas
square = ->(n) { n * n }
puts [1, 2, 3].map(&square)

Advanced Ruby Features

Powerful features that make Ruby expressive and flexible

Classes and Objects

Ruby is a pure object-oriented language where everything is an object.

class Person
  # Constructor
  def initialize(name, age)
    @name = name  # Instance variable
    @age = age
  end
  
  # Instance method
  def introduce
    puts "Hi, I'm #{@name} and I'm #{@age} years old."
  end
  
  # Getter method
  def name
    @name
  end
  
  # Setter method
  def name=(new_name)
    @name = new_name
  end
  
  # Class method
  def self.species
    "Homo sapiens"
  end
  
  # Attribute accessors (creates getters/setters)
  attr_accessor :age
  attr_reader :id
  attr_writer :secret
end

# Create an object
person = Person.new("Alice", 30)
person.introduce
person.name = "Alicia"  # Call setter method
puts person.name        # Call getter method

# Class method call
puts Person.species

# Inheritance
class Employee < Person
  def initialize(name, age, position)
    super(name, age)
    @position = position
  end
  
  def introduce
    super
    puts "I work as a #{@position}."
  end
end

employee = Employee.new("Bob", 40, "Developer")
employee.introduce

Attribute Accessors: Ruby provides attr_accessor (getter + setter), attr_reader (getter only), and attr_writer (setter only) to automatically generate getter and setter methods for instance variables.

Object-Oriented Ruby Explained

Ruby's object-oriented features are central to the language:

Key Concepts
  • Class: Blueprint for creating objects
  • Object: Instance of a class
  • Instance variables: @variable - unique to each object
  • Class variables: @@variable - shared by all instances
  • Instance methods: Called on objects
  • Class methods: Called on the class itself
Inheritance

Ruby uses single inheritance (one superclass) with mixins for additional functionality:

class Animal
  def speak
    "Some sound"
  end
end

class Dog < Animal
  def speak
    "Woof!"
  end
end

dog = Dog.new
puts dog.speak # => "Woof!"
Best Practices
  • Follow the single responsibility principle
  • Keep classes small and focused
  • Use composition over inheritance when possible
  • Use modules for shared behavior
  • Name classes with nouns and methods with verbs

Modules and Mixins

Modules are collections of methods and constants that can be included in classes.

module Greetable
  def greet
    puts "Hello from #{self.class}"
  end
end

class Person
  include Greetable
end

class Robot
  include Greetable
end

Person.new.greet  # => "Hello from Person"
Robot.new.greet   # => "Hello from Robot"

# Namespacing with modules
module Math
  PI = 3.14159
  
  def self.square(x)
    x * x
  end
end

puts Math::PI
puts Math.square(5)

# Module as container
module MyLibrary
  class Book
    # ...
  end
  
  class Author
    # ...
  end
end

book = MyLibrary::Book.new

Modules vs. Classes: Modules cannot be instantiated or subclassed. They're used for namespacing and mixins. Classes can be instantiated and follow single inheritance, but can include multiple modules.

Modules Explained

Modules serve two main purposes in Ruby:

1. Mixins

Share functionality between classes:

  • include: Adds methods as instance methods
  • extend: Adds methods as class methods
  • prepend: Inserts module before class in method lookup
2. Namespacing

Organize code and prevent naming conflicts:

module Payment
  class Invoice
    # ...
  end
  
  class Receipt
    # ...
  end
end

invoice = Payment::Invoice.new
Common Ruby Modules
  • Enumerable: Collection methods (each, map, etc.)
  • Comparable: Comparison methods (<, >, etc.)
  • Kernel: Core methods available everywhere
When to Use Modules
  • When you need to share behavior between classes
  • When you need to organize related classes
  • When you want to add functionality without inheritance

Metaprogramming

Ruby allows you to write code that writes code, known as metaprogramming.

# Dynamically define methods
class Person
  %w[name age email].each do |attribute|
    define_method(attribute) do
      instance_variable_get("@#{attribute}")
    end
    
    define_method("#{attribute}=") do |value|
      instance_variable_set("@#{attribute}", value)
    end
  end
end

person = Person.new
person.name = "Alice"
puts person.name # => "Alice"

# method_missing
class Roman
  def method_missing(name, *args)
    roman = name.to_s
    roman.gsub!("IV", "IIII")
    roman.gsub!("IX", "VIIII")
    roman.gsub!("XL", "XXXX")
    roman.gsub!("XC", "LXXXX")
    
    (roman.count("I") +
     roman.count("V") * 5 +
     roman.count("X") * 10 +
     roman.count("L") * 50 +
     roman.count("C") * 100)
  end
end

r = Roman.new
puts r.iv # => 4
puts r.ix # => 9
puts r.xl # => 40

# send method
class Test
  def hello(name)
    puts "Hello, #{name}!"
  end
end

t = Test.new
t.send(:hello, "Alice") # => "Hello, Alice!"

# eval and friends (use with caution!)
code = "2 + 2"
puts eval(code) # => 4

# Class macros
class Vehicle
  def self.types(*types)
    types.each do |type|
      define_method("#{type}?") do
        @type == type
      end
    end
  end
  
  types :car, :truck, :bus
end

v = Vehicle.new
v.instance_variable_set("@type", :car)
puts v.car? # => true
puts v.truck? # => false

Metaprogramming Caution: While powerful, metaprogramming can make code harder to understand and debug. Use it judiciously and document thoroughly. Always prefer clear, explicit code over clever metaprogramming when possible.

Metaprogramming Explained

Metaprogramming is writing code that writes code. Here's what beginners should know:

Common Techniques
  • define_method: Create methods dynamically
  • method_missing: Handle calls to undefined methods
  • send: Call methods dynamically
  • eval: Execute strings as code (use carefully)
  • Class macros: Methods that define other methods
Practical Uses
# Dynamic finders (like ActiveRecord)
class Record
  def self.find_by(attribute, value)
    # ... database lookup ...
  end
  
  def self.method_missing(name, *args)
    if name.to_s.start_with?("find_by_")
      attribute = name.to_s.split("find_by_").last
      find_by(attribute, args.first)
    else
      super
    end
  end
end

# Record.find_by_name("Alice") would call find_by(:name, "Alice")

# Configuration DSL
class Config
  def self.setting(name)
    define_method(name) do
      instance_variable_get("@#{name}")
    end
    
    define_method("#{name}=") do |value|
      instance_variable_set("@#{name}", value)
    end
  end
end

class AppConfig < Config
  setting :title
  setting :version
end

config = AppConfig.new
config.title = "My App"
puts config.title # => "My App"
When to Use Metaprogramming
  • When you need to reduce repetitive code
  • When creating domain-specific languages (DSLs)
  • When building flexible APIs
  • When implementing dynamic behavior
When to Avoid
  • When simple methods would suffice
  • When clarity is more important than cleverness
  • When performance is critical
