What is CSS?

The foundation of web styling and design

The Styling Language of the Web

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML or XML. It controls how elements are displayed on screen, on paper, in speech, or on other media.

Key Features of CSS

  • Separation of concerns: CSS separates content (HTML) from presentation (styling)
  • Cascading nature: Multiple style rules can apply to the same element
  • Responsive design: Adapt layouts to different screen sizes
  • Animation capabilities: Create movement and transitions
  • Powerful selectors: Precisely target elements to style
/* This is a simple CSS rule */
h1 {
    color: blue;
    font-size: 2em;
    margin-bottom: 20px;
}

Did you know? The "cascading" in CSS refers to how styles can inherit and override each other based on specificity and source order. Styles can come from multiple sources (browser defaults, user stylesheets, author stylesheets), and CSS determines which styles take precedence.

Absolute Beginner's Guide

If you're completely new to CSS, here's what you need to know to get started:

  1. CSS works with HTML: You write CSS rules that target HTML elements to style them
  2. Three ways to add CSS:
    • Inline styles (directly in HTML elements)
    • Internal stylesheet (in the <style> tag)
    • External stylesheet (separate .css file linked to HTML)
  3. Basic syntax: A CSS rule has a selector and declaration block with properties and values
  4. Start simple: Begin with basic properties like color, font-size, and margin

First Steps Exercise: Try creating a simple HTML file and add some CSS to change the text color and background. Experiment with different selectors like element names, classes, and IDs.

Basic CSS Properties

Essential styling properties every developer should know

Text Styling Properties

CSS provides numerous properties to style text content. Here are some of the most fundamental ones:

/* Text styling example */
.article-title {
    font-family: 'Georgia', serif;  /* Font family */
    font-size: 2.5rem;             /* Font size */
    font-weight: 700;              /* Boldness */
    line-height: 1.5;              /* Line spacing */
    color: #2d3748;                /* Text color */
    text-align: center;            /* Alignment */
    text-transform: uppercase;     /* Text transformation */
    text-decoration: underline;    /* Underline */
    letter-spacing: 1px;           /* Character spacing */
    word-spacing: 2px;             /* Space between words */
}

Comparison of unstyled vs styled text

Font Stack Best Practices

Always specify a font stack (multiple font options) to ensure fallbacks if your preferred font isn't available. Example: font-family: 'Helvetica Neue', Arial, sans-serif;

Text Styling Tips for Beginners

  • Start with a base font size of 16px (1rem) for body text
  • Use relative units (em, rem) for font sizes to support accessibility
  • Limit your font choices to 2-3 fonts per page for consistency
  • Ensure sufficient contrast between text and background colors
  • Use line-height between 1.4-1.6 for optimal readability

Color and Background Properties

CSS offers multiple ways to specify colors and style backgrounds:

/* Color and background example */
.header {
    color: white;                  /* Text color */
    background-color: #2563eb;      /* Background color */
    
    /* Gradient background */
    background-image: linear-gradient(to right, #2563eb, #7c3aed);
    
    /* Multiple background images */
    background-image: url('pattern.png'), linear-gradient(to bottom, #2563eb, #7c3aed);
    background-blend-mode: overlay;
    
    /* Background control */
    background-repeat: no-repeat;
    background-position: center;
    background-size: cover;
    background-attachment: fixed;   /* Parallax effect */
}

Color Formats: CSS supports several color formats: HEX (#RRGGBB), RGB (rgb(255,0,0)), HSL (hsl(0,100%,50%)), and named colors (red, blue, etc.). Modern CSS also supports alpha transparency with RGBA and HSLA.

Beginner Tip

Use CSS variables for colors to make your stylesheets more maintainable. Define them in the :root selector and reference them with var(--variable-name).

Color Theory Basics

When starting with CSS colors, keep these principles in mind:

  • Color harmony: Use complementary or analogous color schemes
  • Contrast: Ensure text is readable against backgrounds
  • Consistency: Limit your palette to 3-5 main colors
  • Accessibility: Check color contrast ratios (minimum 4.5:1 for normal text)

Box Model Properties

The CSS box model is fundamental to understanding layout. Every element is a rectangular box with these properties:

/* Box model example */
.box {
    width: 300px;
    height: 200px;
    padding: 20px;          /* Inner spacing */
    border: 5px solid black; /* Border around padding */
    margin: 30px;           /* Outer spacing */
    
    /* Box sizing - includes padding/border in width/height */
    box-sizing: border-box;
    
    /* Border radius for rounded corners */
    border-radius: 10px;
    
    /* Box shadow */
    box-shadow: 0 5px 15px rgba(0,0,0,0.1);
}

Warning: The default box-sizing: content-box can cause layout issues. Most developers recommend setting box-sizing: border-box globally for all elements.

Box Model Practice

To understand the box model, try this exercise:

  1. Create a div with some content
  2. Add different padding, border, and margin values
  3. Toggle between box-sizing: content-box and border-box
  4. Use your browser's developer tools to inspect the box model

This hands-on approach will help solidify your understanding of how these properties interact.

Learn More About the Box Model

CSS Box Model

Understanding the foundation of CSS layout

Mastering the Box Model

Every element in CSS is a rectangular box with the following properties:

Box Model Components

  • Content: The actual content (text, images, etc.)
  • Padding: Space between content and border (inside the element)
  • Border: Line surrounding padding and content
  • Margin: Space outside the border (between elements)
/* Box model demonstration */
.box {
    width: 300px;           /* Content width */
    height: 200px;          /* Content height */
    padding: 20px;          /* Inner spacing */
    border: 5px solid #000; /* Visible border */
    margin: 30px;           /* Outer spacing */
    
    /* Total width calculation: */
    /* content + padding + border = 300 + 40 + 10 = 350px */
    /* (margin adds to the space between elements) */
}

Box-Sizing Property: The box-sizing property changes how width and height are calculated. content-box (default) only includes content, while border-box includes padding and border in the dimensions.

Practical Box Model Tips

When working with the box model:

  • Use consistent spacing units (like multiples of 8px)
  • Start with margin: 0 and padding: 0 in your CSS reset
  • Use browser dev tools to visualize the box model
  • Remember that margins can collapse vertically
  • Consider using CSS variables for consistent spacing

Margin Collapsing

One of the most important concepts to understand about the box model is margin collapsing:

When two vertical margins meet, they collapse to form a single margin whose size is the largest of the individual margins. This only happens with vertical margins (top and bottom), not horizontal (left and right) margins.

/* Margin collapsing example */
.box1 {
    margin-bottom: 30px;
}

.box2 {
    margin-top: 20px;
}

/* The space between .box1 and .box2 will be 30px (not 50px) */
/* because the margins collapse into the larger value */

Tip: To prevent margin collapsing, you can use padding instead, add a border, or use flexbox/grid layout which doesn't have margin collapsing.

Common Box Model Mistakes

Beginners often struggle with:

  • Forgetting to reset default margins/padding
  • Not understanding margin collapsing
  • Using width: 100% with padding (causing overflow)
  • Mixing absolute and relative units inconsistently

Practice identifying and fixing these issues in your projects.

CSS Positioning

Control the layout and placement of elements

Position Property

The position property specifies the type of positioning method used for an element:

Position Values

  • static: Default. Elements render in order of document flow
  • relative: Positioned relative to its normal position
  • absolute: Positioned relative to nearest positioned ancestor
  • fixed: Positioned relative to viewport (stays on screen during scroll)
  • sticky: Hybrid of relative and fixed positioning
/* Positioning examples */
.relative-box {
    position: relative;
    top: 20px;
    left: 30px;
}

.absolute-box {
    position: absolute;
    top: 0;
    right: 0;
}

.fixed-box {
    position: fixed;
    bottom: 20px;
    right: 20px;
}

.sticky-header {
    position: sticky;
    top: 0;
}

Warning: Absolutely positioned elements are removed from the normal document flow, which can cause layout issues if overused. Use them judiciously.

Positioning Practice

To master CSS positioning:

  1. Create a container with several child elements
  2. Experiment with each position value
  3. Notice how each affects document flow
  4. Combine with top/right/bottom/left properties
  5. Try creating a sticky header and fixed footer

Z-Index and Stacking Context

When elements overlap, the z-index property determines which appears on top:

/* Z-index example */
.modal {
    position: fixed;
    z-index: 1000; /* Higher z-index appears on top */
}

.dropdown {
    position: absolute;
    z-index: 100;
}

.tooltip {
    position: absolute;
    z-index: 200;
}

Stacking Context: Z-index only works on positioned elements (anything except static). Each stacking context is like a separate "layer" where z-index values are compared only within that context.

Beginner Tip

If z-index isn't working as expected, check if any parent element creates a new stacking context (common causes: opacity < 1, transform, filter, or isolation properties).

Understanding Stacking Order

Elements stack in this order (from bottom to top):

  1. Non-positioned elements in normal flow
  2. Positioned elements (in order of appearance)
  3. Elements with z-index ≥ 1

Remember that z-index only works within the same stacking context.

Flexbox Layout

The modern way to create flexible one-dimensional layouts

Flexbox Fundamentals

Flexbox (Flexible Box Layout) is a one-dimensional layout method for arranging items in rows or columns.

Flexbox Concepts

  • Flex container: Parent element with display: flex
  • Flex items: Direct children of the flex container
  • Main axis: Primary axis (row or column) defined by flex-direction
  • Cross axis: Perpendicular to main axis
/* Flexbox container */
.container {
    display: flex;           /* Activate flexbox */
    flex-direction: row;     /* Main axis direction (row|row-reverse|column|column-reverse) */
    justify-content: center; /* Main axis alignment */
    align-items: center;     /* Cross axis alignment */
    gap: 20px;              /* Space between items */
    flex-wrap: wrap;        /* Allow items to wrap */
}

/* Flex items */
.item {
    flex: 1;               /* Grow/shrink proportionally */
    min-width: 100px;      /* Minimum size */
}

Flexbox Learning Path

To learn Flexbox effectively:

  1. Start with display: flex on a container
  2. Experiment with flex-direction
  3. Try different justify-content values
  4. Explore align-items and align-self
  5. Learn about flex-grow, flex-shrink, and flex-basis
  6. Practice creating common layouts (navbars, cards, etc.)

Flexbox Properties

Flexbox provides powerful properties to control layout:

/* Container properties */
.flex-container {
    display: flex;
    flex-direction: row; /* or column */
    justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly;
    align-items: stretch | flex-start | flex-end | center | baseline;
    align-content: flex-start | flex-end | center | space-between | space-around | stretch;
    flex-wrap: nowrap | wrap | wrap-reverse;
    gap: 10px;
}

/* Item properties */
.flex-item {
    order: 0; /* Change visual order */
    flex-grow: 0; /* Ability to grow */
    flex-shrink: 1; /* Ability to shrink */
    flex-basis: auto; /* Default size */
    align-self: auto | flex-start | flex-end | center | baseline | stretch;
}

Shorthand: The flex property is shorthand for flex-grow, flex-shrink, and flex-basis. Common values: flex: 1 (grow equally), flex: 0 0 200px (fixed width), flex: 1 0 auto (grow but don't shrink below content size).

Flexbox Cheat Sheet

Quick reference for common Flexbox needs:

  • Center items: justify-content: center; align-items: center;
  • Space evenly: justify-content: space-between;
  • Equal height columns: align-items: stretch; (default)
  • Mobile menu: flex-direction: column;
  • Responsive grid: flex-wrap: wrap; with percentage widths

Practical Flexbox Examples

Here are some common use cases for Flexbox:

/* Navigation bar */
.nav {
    display: flex;
    justify-content: space-between;
    align-items: center;
    padding: 1rem;
}

/* Card layout */
.card-container {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
}

.card {
    flex: 1 0 300px; /* Grow, don't shrink, base 300px */
}

/* Centering content */
.center-container {
    display: flex;
    justify-content: center;
    align-items: center;
    height: 100vh;
}

/* Holy Grail Layout */
.holy-grail {
    display: flex;
    flex-direction: column;
    min-height: 100vh;
}

.holy-grail main {
    display: flex;
    flex: 1;
}

.holy-grail .content {
    flex: 1;
}

.holy-grail .sidebar {
    flex: 0 0 250px;
}

Browser Support: Flexbox is supported in all modern browsers. For very old browsers, consider using feature queries (@supports) to provide fallbacks.

CSS Grid Layout

The most powerful two-dimensional layout system

Grid Fundamentals

CSS Grid is a two-dimensional layout system that handles both rows and columns.

Grid Concepts

  • Grid container: Parent element with display: grid
  • Grid items: Direct children of the grid container
  • Grid lines: The dividing lines that make up the grid
  • Grid tracks: The space between two adjacent grid lines (rows or columns)
  • Grid cells: A single unit of the grid
  • Grid areas: Rectangular space surrounded by four grid lines
/* Basic grid example */
.container {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr; /* Three equal columns */
    grid-template-rows: 100px auto 100px; /* Three rows */
    gap: 20px; /* Space between items */
}

/* Using grid areas */
.layout {
    display: grid;
    grid-template-columns: 1fr 3fr 1fr;
    grid-template-rows: auto 1fr auto;
    grid-template-areas:
        "header header header"
        "sidebar main aside"
        "footer footer footer";
    min-height: 100vh;
}

.header { grid-area: header; }
.main   { grid-area: main; }
.sidebar { grid-area: sidebar; }
.footer { grid-area: footer; }

Grid Learning Path

To master CSS Grid:

  1. Start with simple grid definitions
  2. Practice with grid-template-columns and grid-template-rows
  3. Learn about the fr unit
  4. Experiment with gap for spacing
  5. Try grid line placement
  6. Move to named grid areas
  7. Explore responsive techniques

Grid Properties

CSS Grid provides an extensive set of properties for creating complex layouts:

/* Container properties */
.grid-container {
    display: grid | inline-grid;
    grid-template-columns: track-size | line-name track-size ...;
    grid-template-rows: track-size | line-name track-size ...;
    grid-template-areas: grid-area-names | none;
    grid-column-gap: line-size;
    grid-row-gap: line-size;
    justify-items: start | end | center | stretch;
    align-items: start | end | center | stretch;
    place-items: align-items justify-items;
    justify-content: start | end | center | stretch | space-around | space-between | space-evenly;
    align-content: start | end | center | stretch | space-around | space-between | space-evenly;
    grid-auto-columns: track-size;
    grid-auto-rows: track-size;
    grid-auto-flow: row | column | row dense | column dense;
}

/* Item properties */
.grid-item {
    grid-column: start-line / end-line | start-line / span value;
    grid-row: start-line / end-line | start-line / span value;
    grid-area: name | row-start / column-start / row-end / column-end;
    justify-self: start | end | center | stretch;
    align-self: start | end | center | stretch;
    place-self: align-self justify-self;
}

Fr Units: The fr unit represents a fraction of the available space in the grid container. For example, grid-template-columns: 1fr 2fr creates two columns where the second is twice as wide as the first.

Grid Cheat Sheet

Quick reference for common Grid needs:

  • 12-column grid: grid-template-columns: repeat(12, 1fr);
  • Responsive grid: grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  • Center items: place-items: center;
  • Gap control: gap: 20px; or row-gap/column-gap
  • Explicit placement: grid-column: 1 / 3; or grid-area: header;

Practical Grid Examples

CSS Grid excels at creating complex layouts with minimal code:

/* Responsive gallery */
.gallery {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
    gap: 15px;
}

/* Magazine layout */
.magazine {
    display: grid;
    grid-template-columns: repeat(12, 1fr);
    gap: 20px;
}

.feature {
    grid-column: 1 / -1;
}

.article {
    grid-column: span 4;
}

.sidebar {
    grid-column: span 3;
}

/* Holy Grail Layout */
.holy-grail {
    display: grid;
    grid-template:
        "header header header" auto
        "nav content aside" 1fr
        "footer footer footer" auto
        / auto 1fr auto;
    min-height: 100vh;
}

.header { grid-area: header; }
.nav { grid-area: nav; }
.content { grid-area: content; }
.aside { grid-area: aside; }
.footer { grid-area: footer; }

Grid vs Flexbox: Use Grid for overall page layout (2D) and Flexbox for components within those sections (1D). They work beautifully together!

Responsive Design

Create websites that work on any device

Media Queries

Media queries allow you to apply CSS styles depending on device characteristics:

/* Basic media query */
@media screen and (max-width: 768px) {
    .container {
        flex-direction: column;
    }
    
    .sidebar {
        display: none;
    }
}

/* Common breakpoints */
/* Small devices (phones) */
@media (max-width: 576px) { ... }

/* Medium devices (tablets) */
@media (min-width: 576px) and (max-width: 992px) { ... }

/* Large devices (desktops) */
@media (min-width: 992px) { ... }

/* Orientation */
@media (orientation: portrait) { ... }
@media (orientation: landscape) { ... }

/* Pixel density */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) { 
    /* High resolution styles */
}

Mobile-First Approach: Start with styles for mobile devices (small screens) and then add media queries to adjust for larger screens. This is often more efficient than the opposite approach.

Responsive Design Tips

For beginners implementing responsive design:

  • Always include the viewport meta tag
  • Start with mobile styles first
  • Use relative units (rem, em, %) instead of pixels
  • Test on actual devices when possible
  • Use Chrome DevTools device toolbar for testing
  • Consider using CSS Grid and Flexbox for layouts

Responsive Units

CSS provides several units that work well for responsive design:

Relative Units

  • %: Percentage of parent element
  • vw/vh: Percentage of viewport width/height
  • rem: Relative to root element's font-size
  • em: Relative to current element's font-size
  • ch: Width of the "0" character
  • fr: Fractional unit (Grid only)
/* Responsive typography */
html {
    font-size: 16px;
}

h1 {
    font-size: 2rem; /* 32px */
}

@media (max-width: 768px) {
    html {
        font-size: 14px;
    }
    /* h1 now equals 28px */
}

/* Fluid spacing */
.container {
    padding: 2vw;
}

/* Responsive grid */
.grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
    gap: 1rem;
}

Beginner Tip

Use rem units for font sizes and spacing to create a consistent, scalable layout. Reserve px for borders and other cases where you need precise control.

Unit Selection Guide

When to use which units:

  • rem: Font sizes, padding, margin (for scalability)
  • em: When you need sizing relative to parent font size
  • %: Widths, heights relative to parent
  • vw/vh: Full-viewport elements (hero sections)
  • px: Borders, shadows, precise positioning
  • fr: Grid layouts

Responsive Images

Ensure images adapt to different screen sizes and resolutions:

<!-- Responsive image -->
<img src="image.jpg" alt="Description" 
     srcset="image-small.jpg 480w,
             image-medium.jpg 768w,
             image-large.jpg 1200w"
     sizes="(max-width: 600px) 480px,
            (max-width: 1000px) 768px,
            1200px">

<!-- Art direction -->
<picture>
    <source media="(max-width: 768px)" srcset="mobile.jpg">
    <source media="(min-width: 769px)" srcset="desktop.jpg">
    <img src="desktop.jpg" alt="Description">
</picture>

/* CSS solution */
.responsive-img {
    max-width: 100%;
    height: auto;
}

Performance Matters: Always optimize images for the web. Use modern formats like WebP, compress images, and serve appropriately sized versions for each device.

Responsive Images Checklist

For beginners implementing responsive images:

  1. Always include the alt attribute
  2. Use max-width: 100% for simple responsive images
  3. Consider srcset for resolution switching
  4. Use <picture> for art direction
  5. Compress images (aim for <100KB where possible)
  6. Lazy load images below the fold

Transitions & Animations

Bring your interfaces to life with motion

CSS Transitions

Transitions allow you to smoothly animate property changes over a specified duration:

/* Basic transition */
.button {
    background-color: #2563eb;
    transition: background-color 0.3s ease;
}

.button:hover {
    background-color: #1e40af;
}

/* Multiple properties */
.card {
    transition: transform 0.3s ease, box-shadow 0.3s ease;
}

.card:hover {
    transform: translateY(-5px);
    box-shadow: 0 10px 20px rgba(0,0,0,0.1);
}

/* Transition shorthand */
.element {
    transition: property duration timing-function delay;
    
    /* Example */
    transition: all 0.3s cubic-bezier(0.25, 0.1, 0.25, 1) 0.1s;
}

Transition Properties: Not all CSS properties can be transitioned. Generally, properties that accept numeric values (colors, sizes, positions) can be animated, while properties like display cannot.

Transition Practice

To practice CSS transitions:

  1. Create a button with hover effects
  2. Add transitions to color changes
  3. Experiment with different timing functions
  4. Try transitioning multiple properties
  5. Add delays to create sequenced animations

CSS Animations

For more complex animations, CSS provides the @keyframes rule:

/* Keyframes animation */
@keyframes slideIn {
    from {
        transform: translateX(-100%);
        opacity: 0;
    }
    to {
        transform: translateX(0);
        opacity: 1;
    }
}

/* Applying animation */
.slide-in {
    animation: slideIn 0.5s ease-out forwards;
}

/* Animation shorthand */
.element {
    animation: name duration timing-function delay iteration-count direction fill-mode;
    
    /* Example */
    animation: pulse 2s ease-in-out infinite alternate;
}

@keyframes pulse {
    0% { transform: scale(1); }
    100% { transform: scale(1.05); }
}

Animation Properties

  • animation-name: The @keyframes name
  • animation-duration: How long the animation lasts
  • animation-timing-function: Speed curve (ease, linear, etc.)
  • animation-delay: Time before animation starts
  • animation-iteration-count: How many times to play (number or infinite)
  • animation-direction: normal, reverse, alternate, alternate-reverse
  • animation-fill-mode: What values apply before/after animation
  • animation-play-state: running or paused

Animation Ideas for Beginners

Simple animations to practice:

  • Fade-in effects for page elements
  • Loading spinners
  • Hover animations for buttons/cards
  • Slide-in menus
  • Attention-grabbing call-to-action elements

Performance Considerations

Animations can impact performance if not implemented carefully:

Best Practices

  • Use transform and opacity: These properties can be hardware-accelerated
  • Avoid animating width/height: Use transform: scale() instead
  • Limit simultaneous animations: Too many animations can cause jank
  • Use will-change: Hint to browser which properties will change
  • Test on mobile: Mobile devices have less processing power
/* Performant animations */
.performant-box {
    transform: translateZ(0); /* Trigger hardware acceleration */
    will-change: transform, opacity;
}

/* Bad practice - may cause layout recalculations */
.slow-animation {
    animation: changeWidth 2s infinite;
}

@keyframes changeWidth {
    from { width: 100px; }
    to { width: 200px; }
}

Accessibility: Some users prefer reduced motion due to vestibular disorders. Always respect the prefers-reduced-motion media query and provide alternatives.

Animation Performance Tips

For beginners creating animations:

  • Start simple - don't over-animate
  • Test on lower-powered devices
  • Use the Chrome Performance tab to check for jank
  • Prefer transforms/opacity over layout-changing properties
  • Consider using will-change for complex animations

CSS Variables

Custom properties for more maintainable stylesheets

Custom Properties Basics

CSS variables (custom properties) allow you to store values for reuse throughout your stylesheet:

/* Defining variables */
:root {
    --primary-color: #2563eb;
    --secondary-color: #7c3aed;
    --spacing-unit: 1rem;
    --max-width: 1200px;
    --box-shadow: 0 5px 15px rgba(0,0,0,0.1);
}

/* Using variables */
.button {
    background-color: var(--primary-color);
    padding: var(--spacing-unit) calc(var(--spacing-unit) * 2);
}

.card {
    max-width: var(--max-width);
    box-shadow: var(--box-shadow);
}

/* Fallback values */
.element {
    color: var(--undefined-var, #000); /* Falls back to #000 */
}

Naming Convention: Use descriptive names for variables. Common patterns include --color-primary, --spacing-medium, --font-heading. Group related variables together in your :root declaration.

CSS Variables Practice

To get started with CSS variables:

  1. Identify values you reuse frequently (colors, spacing)
  2. Define them as variables in :root
  3. Replace hardcoded values with variables
  4. Experiment with changing values in one place
  5. Try using variables for calculations

Dynamic Theming

CSS variables can be updated with JavaScript, enabling dynamic theming:

/* CSS */
:root {
    --bg-color: #fff;
    --text-color: #333;
}

body {
    background-color: var(--bg-color);
    color: var(--text-color);
    transition: background-color 0.3s, color 0.3s;
}

.dark-mode {
    --bg-color: #1a202c;
    --text-color: #f7fafc;
}

/* JavaScript */
const toggle = document.querySelector('.theme-toggle');
toggle.addEventListener('click', () => {
    document.body.classList.toggle('dark-mode');
});

Browser Support: CSS variables are supported in all modern browsers. For Internet Explorer, you'll need to provide fallback values or use a polyfill.

Theming Tips

When implementing themes with CSS variables:

  • Define all theme-related values as variables
  • Use semantic names (--text-color, --bg-color)
  • Provide proper contrast in all themes
  • Consider system preference (prefers-color-scheme)
  • Store user preference in localStorage

Advanced Techniques

CSS variables can be used for more than just simple value storage:

/* Calculations */
:root {
    --base-size: 16px;
    --header-size: calc(var(--base-size) * 2.5);
}

/* Media queries */
@media (max-width: 768px) {
    :root {
        --base-size: 14px;
    }
}

/* Component-scoped variables */
.alert {
    --alert-bg: #f8d7da;
    --alert-color: #721c24;
    background-color: var(--alert-bg);
    color: var(--alert-color);
}

.alert.success {
    --alert-bg: #d4edda;
    --alert-color: #155724;
}

/* Manipulating with JavaScript */
element.style.setProperty('--primary-color', newColor);

Beginner Tip

Start by converting your most reused values (colors, spacing, font sizes) into variables. As you become comfortable, explore more advanced uses like theming and dynamic updates.

Creative Uses of CSS Variables

Beyond basic theming, CSS variables can be used for:

  • Responsive typography scaling
  • Interactive components (sliders, color pickers)
  • Animation control (timing, distances)
  • Theme variations (light/dark/high contrast)
  • Dynamic grid layouts

Advanced Layout Techniques

Modern CSS features for complex layouts

Multi-column Layout

The CSS Multi-column Layout module allows content to flow between columns:

/* Basic multi-column layout */
.article {
    column-count: 3;
    column-gap: 2em;
    column-rule: 1px solid #ddd;
}

/* Controlling column breaks */
h2 {
    column-span: all; /* Span all columns */
    break-after: avoid; /* Avoid breaking after */
}

/* Responsive columns */
@media (max-width: 768px) {
    .article {
        column-count: 2;
    }
}

@media (max-width: 480px) {
    .article {
        column-count: 1;
    }
}

Use Cases: Multi-column is perfect for newspaper-style layouts, long lists, or any content that benefits from a vertical reading flow. It's especially useful for text-heavy pages.

Multi-column Tips

When using multi-column layout:

  • Consider readability - don't make columns too narrow
  • Use column-gap for proper spacing
  • Control breaks with break-inside and break-after
  • Use column-span for elements that should span all columns
  • Test on mobile - you may need to reduce columns

CSS Shapes

The CSS Shapes module allows text to flow around custom shapes:

/* Circle shape */
.circle {
    width: 200px;
    height: 200px;
    background-color: #2563eb;
    float: left;
    shape-outside: circle(50%);
    clip-path: circle(50%);
    margin-right: 20px;
}

/* Polygon shape */
.polygon {
    width: 200px;
    height: 200px;
    background-color: #7c3aed;
    float: right;
    shape-outside: polygon(0 0, 100% 0, 100% 100%, 0 50%);
    clip-path: polygon(0 0, 100% 0, 100% 100%, 0 50%);
    margin-left: 20px;
}

Browser Support: CSS Shapes work best when combined with floats. For full support, check compatibility as some newer features might not be available in all browsers.

Shapes Learning Path

To get started with CSS Shapes:

  1. Start with basic shapes (circle, ellipse)
  2. Experiment with polygon shapes
  3. Combine with float for text wrapping
  4. Use clip-path to visually match the shape
  5. Try using images with alpha channels as shapes

CSS Subgrid

Subgrid allows grid items to inherit the grid of their parent, enabling more complex layouts:

/* Parent grid */
.grid {
    display: grid;
    grid-template-columns: repeat(12, 1fr);
    gap: 20px;
}

/* Child with subgrid */
.grid-item {
    display: grid;
    grid-column: span 6;
    grid-template-rows: subgrid;
    grid-row: span 3;
}

/* Practical example */
.card-grid {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
    gap: 30px;
}

.card {
    display: grid;
    grid-template-rows: auto 1fr auto;
    gap: 15px;
}

.card-header, .card-footer {
    grid-column: 1 / -1;
}

Browser Support: Subgrid is relatively new (as of 2023) and may not be supported in all browsers. Always check current compatibility and provide fallbacks where needed.

Beginner Tip

Start with mastering basic Grid before diving into Subgrid. The concepts build on each other, and you'll better appreciate Subgrid's benefits after working with regular Grid.

When to Use Subgrid

Subgrid is particularly useful for:

  • Card layouts with aligned content
  • Forms with complex layouts
  • Dashboards with nested components
  • Any design where child elements need to align with parent grid

For simpler layouts, regular Grid might be sufficient.
